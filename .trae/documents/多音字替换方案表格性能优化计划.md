# 多音字替换方案表格性能优化计划

## 问题分析

通过代码分析，发现以下性能瓶颈：

### 1. DataGridView 自动尺寸计算问题（主要瓶颈）
**位置**: `PolyphonicReplacePanel.Designer.cs` 第324-325行
```csharp
this.dgv_多音字方案.AutoSizeColumnsMode = DataGridViewAutoSizeColumnsMode.AllCells;
this.dgv_多音字方案.AutoSizeRowsMode = DataGridViewAutoSizeRowsMode.AllCells;
```
**问题**: `AllCells` 模式会在每次数据变化时计算所有单元格的尺寸，数据量大时非常耗时。

### 2. 频繁的 UI 刷新调用
**位置**: `PolyphonicReplacePanel.cs` 多处事件处理
- `dgv_多音字方案_CellEndEdit` → 调用 `refresh多音字替换Ui()`
- `dgv_多音字方案_RowsAdded` → 调用 `refresh多音字替换Ui()`
- `dgv_多音字方案_UserDeletedRow` → 调用 `refresh多音字替换Ui()`
- `tb_最终文本_TextChanged` → 调用 `refresh多音字替换Ui()`

**问题**: 每次编辑、添加、删除行都会触发完整的 UI 刷新，而实际上只需要更新按钮状态。

### 3. DataTable 转换开销
**位置**: `PolyphonicItem.cs` 第139-162行、第169-194行
- `ConvertToDataTable()` 和 `ConvertFromDataTable()` 每次保存/还原都会重新创建 DataTable
- 每次转换都要遍历所有行和列

### 4. 不必要的控件状态操作
**位置**: `PolyphonicReplacePanel.cs` 第291-391行 `refresh多音字替换Ui()` 方法
- 每次刷新都操作文本框的选择状态，即使文本框未获得焦点
- 每次刷新都重新计算按钮的 Enabled 状态

---

## 优化方案

### 方案一：优化 DataGridView 自动尺寸模式（高优先级）

**修改内容**:
1. 将 `AutoSizeColumnsMode` 从 `AllCells` 改为 `DisplayedCells`（只计算可见单元格）
2. 将 `AutoSizeRowsMode` 从 `AllCells` 改为 `DisplayedCells`

**预期效果**: 大幅减少尺寸计算时间，尤其是数据量大时。

**修改文件**: `PolyphonicReplacePanel.Designer.cs`

---

### 方案二：减少不必要的 UI 刷新（高优先级）

**修改内容**:
1. 创建轻量级的 `refreshButtonStates()` 方法，只更新按钮状态
2. 在 `CellEndEdit`、`RowsAdded`、`UserDeletedRow` 事件中只调用轻量级刷新
3. 保留完整刷新用于关键操作（如开始替换、应用结果等）

**预期效果**: 减少不必要的控件操作，提升响应速度。

**修改文件**: `PolyphonicReplacePanel.cs`

---

### 方案三：优化 refresh多音字替换Ui 方法（中优先级）

**修改内容**:
1. 只在文本框实际获得焦点时才保存/恢复选择状态
2. 避免重复设置相同的属性值（添加状态检查）
3. 使用 `SuspendLayout()`/`ResumeLayout()` 包裹批量更新

**预期效果**: 减少不必要的控件操作和重绘。

**修改文件**: `PolyphonicReplacePanel.cs`

---

### 方案四：添加 DataGridView 双缓冲（中优先级）

**修改内容**:
1. 创建自定义 DataGridView 控件，启用双缓冲减少闪烁
2. 或使用反射临时启用双缓冲

**预期效果**: 减少绘制闪烁，提升视觉流畅度。

**修改文件**: 新建或修改现有控件

---

### 方案五：延迟刷新策略（低优先级）

**修改内容**:
1. 使用 `Timer` 或 `BeginInvoke` 延迟刷新，避免连续快速操作时的重复刷新
2. 合并多次连续的刷新请求

**预期效果**: 避免快速编辑时的性能问题。

**修改文件**: `PolyphonicReplacePanel.cs`

---

## 实施步骤

1. **第一步**: 修改 DataGridView 的自动尺寸模式（方案一）
2. **第二步**: 创建轻量级刷新方法，减少事件处理中的刷新调用（方案二）
3. **第三步**: 优化 `refresh多音字替换Ui` 方法的实现细节（方案三）
4. **第四步**: 添加双缓冲支持（方案四）
5. **第五步**: 根据测试结果决定是否需要延迟刷新策略（方案五）

---

## 注意事项

- 保持现有的外观和交互方式不变
- 每步优化后进行测试验证
- 优先实施高优先级方案，观察效果后再决定是否需要后续方案
