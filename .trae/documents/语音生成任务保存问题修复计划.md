# 语音生成任务保存问题修复计划

## 一、问题分析

### 1.1 当前语音生成任务的保存机制

语音生成任务**已经保存到系统配置**（`Setting.ini`）中，而非项目文件中：

- **保存**：`save语音生成任务清单Data()` → `Setting.SaveVoiceGenerationTasks(engineId, tasks)` → 保存到 `Setting.ini`
- **加载**：`load语音生成任务清单()` → `Setting.LoadVoiceGenerationTasks(engineId)` → 从 `Setting.ini` 加载

### 1.2 问题根源

在 `MainForm.cs` 的 `clearProjectContent()` 方法中：

```csharp
private void clearProjectContent()
{
    // ...
    _voiceGenerationPanel?.ClearAllTasks();  // ① 清空任务队列
    // ...
    load语音生成PanelData();  // ② 重新加载任务
}
```

问题链路：
1. `ClearAllTasks()` 清空了内存中的任务队列
2. `load语音生成PanelData()` 设置 `CurrentEngineId = getCurrentEngineId()` 后调用 `LoadData()`
3. `LoadData()` 调用 `load语音生成任务清单()`，从系统配置加载任务

**关键问题**：`load语音生成PanelData()` 在 `clearProjectContent()` 中被调用时，`getCurrentEngineId()` 返回的可能是当前选中的引擎ID，而不是之前保存任务时的引擎ID。这导致：
- 如果引擎ID不匹配，会加载错误引擎的任务（或空任务）
- 如果引擎ID匹配但任务已被其他操作覆盖，也会显示错误数据

### 1.3 问题场景

| 场景 | 问题表现 |
|------|----------|
| 新建项目 | `clearProjectContent()` 被调用，任务被清空后重新加载，可能加载到错误引擎的任务 |
| 打开项目 | 同上 |
| 切换TTL引擎 | 任务保存到旧引擎ID，但加载时使用新引擎ID |

## 二、修复方案

### 2.1 核心原则

**语音生成任务是系统级数据，不应受项目操作影响。**

### 2.2 修改点

#### 修改1：`MainForm.cs` - `clearProjectContent()` 方法

**移除** `clearProjectContent()` 中对语音生成任务的清空和重新加载操作：

```csharp
private void clearProjectContent()
{
    getProjectFile()?.ClearAllData();
    _textSplitPanel?.ClearPanel();
    _polyphonicReplacePanel?.ClearPanel();
    _roleEmotionPanel?.ClearData();
    _voicePreprocessPanel?.ClearFinalText();
    // 移除: _voiceGenerationPanel?.ClearAllTasks();  // 不再清空语音生成任务
    _roleMappingPanel?.ClearCachedRoleNames();

    load角色映射PanelData();
    load多音字替换PanelData();
    load角色和情绪指定PanelData();
    load语音生成预处理PanelData();
    // 移除: load语音生成PanelData();  // 不再重新加载语音生成任务
    _voicePreprocessPanel?.LoadDefaultSpeakerSettings();
}
```

#### 修改2：`MainForm.cs` - `loadProjectFile()` 方法

**移除** `loadProjectFile()` 中对语音生成任务的加载：

```csharp
private void loadProjectFile(string filePath)
{
    if (string.IsNullOrWhiteSpace(filePath) == false)
    {
        try
        {
            _currentProjectFilePath = filePath;
            _projectFile = new ProjectFile(_currentProjectFilePath);

            load角色映射PanelData();
            load文本拆分();
            load多音字替换PanelData();
            load角色和情绪指定PanelData();
            load语音生成预处理PanelData();
            // 移除: load语音生成PanelData();  // 项目加载不影响语音生成任务

            _isSaveOnClose = true;
            _isProjectModified = false;
            setProjectMenuEnabled(true);
        }
        // ...
    }
}
```

#### 修改3：`MainForm.cs` - `saveProjectFile()` 方法

**移除** `saveProjectFile()` 中对语音生成任务的保存：

```csharp
private void saveProjectFile()
{
    save文本拆分Data();
    save多音字替换PanelData();
    save角色映射PanelData();
    save角色和情绪指定PanelData();
    save语音生成预处理PanelData();
    // 移除: save语音生成PanelData();  // 项目保存不影响语音生成任务

    getProjectFile()?.Save();
    _isProjectModified = false;
    updateTitleBar();
}
```

#### 修改4：`MainForm.cs` - 程序启动时恢复语音生成任务

在 `MainForm_Shown()` 方法中，**在加载项目文件之后**恢复语音生成任务：

```csharp
private void MainForm_Shown(object sender, EventArgs e)
{
    // ... 现有逻辑 ...

    loadProjectFile(filePath);
    _isInitializing = false;
    _ttlSchemePanel?.StartTtlEngineConnectionIfSelected();

    // 新增：程序启动时恢复语音生成任务
    load语音生成PanelData();

    // ...
}
```

#### 修改5：确认引擎切换逻辑（已正确实现）

经确认，引擎切换逻辑**已正确实现**，无需修改：

```csharp
_ttlSchemePanel.EngineChanged += (s, e) =>
{
    // 先保存旧引擎的数据
    if (!string.IsNullOrEmpty(e.PreviousEngineId))
    {
        // ... 其他面板保存 ...
        _voiceGenerationPanel.CurrentEngineId = e.PreviousEngineId;
        _voiceGenerationPanel.SaveData();  // 保存旧引擎的任务
    }

    // 加载新引擎的数据
    // ... 其他面板加载 ...
    load语音生成PanelData();  // 加载新引擎的任务
};
```

同时，`SaveVoiceGenerationTasksRequested` 事件也在引擎切换前触发：
```csharp
_ttlSchemePanel.SaveVoiceGenerationTasksRequested += (s, e) =>
{
    save语音生成PanelData(getCurrentEngineId());
};
```

**结论**：引擎切换逻辑正确，无需修改。
