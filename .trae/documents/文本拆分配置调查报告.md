## 二、详细说明

### 2.1 保存位置

"拆分长度"值保存在**项目文件**中，而非系统配置。

| 项目 | 详情 |
|------|------|
| 文件格式 | JSON |
| 默认文件名 | `_默认项目_文件.json` |
| 文件位置 | 应用程序根目录 |
| 序列化方式 | `DataContractJsonSerializer` |

### 2.2 与TTL方案的关联关系

**是的，"拆分长度"与TTL方案（引擎）直接关联！**

配置按**引擎ID（engineId）**独立存储，每个TTL引擎可以拥有独立的拆分长度设置。

#### 存储结构示意

```
ProjectData (项目数据)
├── Version                              (版本号)
├── TextSplit_OriginalText               (全局：原始文本，所有引擎共享)
├── TextSplit_FinalText                  (全局：最终文本，所有引擎共享)
└── EngineDataDic                       (按引擎ID区分的字典)
    ├── "engine-id-1"
    │   └── TextSplit_SplitLength        (引擎1的拆分长度：独立)
    ├── "engine-id-2"
    │   └── TextSplit_SplitLength        (引擎2的拆分长度：独立)
    └── ...
```

## 四、配置属性汇总

| 层级 | 属性名 | JSON序列化名 | 默认值 | 说明 |
|------|--------|--------------|--------|------|
| 引擎级 | `TextSplit_SplitLength` | `textSplit_splitLength` | 100 | 每个引擎独立存储 |
| 项目级 | `TextSplit_OriginalText` | `textSplit_originalText` | `""` | 全局共享 |
| 项目级 | `TextSplit_FinalText` | `textSplit_finalText` | `""` | 全局共享 |

## 五、设计特点

1. **隔离性**：每个TTL引擎的拆分长度独立存储，互不影响
2. **持久性**：随项目文件保存，下次打开项目时自动恢复
3. **灵活性**：不同引擎可设置不同的拆分长度值
4. **自动保存**：设置 `IsAutoSave = true` 时，修改后自动保存到文件
