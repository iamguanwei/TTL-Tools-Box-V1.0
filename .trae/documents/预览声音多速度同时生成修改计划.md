# 预览声音多速度同时生成修改计划

## 一、问题分析

### 1.1 当前问题
同一个朗读者，在生成一个速度的预览声音时，如果更改速度后再想生成另一个速度的预览声音，点击按钮后没有反应。

### 1.2 问题原因
在 `TtlSchemePanel.cs` 的 `requestPreviewVoice` 方法中（第1039-1044行），判断是否已有预览任务时，只检查了 `PreviewSourceName`，没有检查 `Speed`：

```csharp
bool alreadyHasPreviewTask = VoiceGenerationTaskQueue?.Tasks.Any(t =>
    t.IsPreview && t.PreviewSourceName == sourceName && t.Status != VoiceGenerationTaskStatus.已完成) ?? false;
```

这导致如果同一个朗读者已经在生成预览声音（不管什么速度），就会阻止创建新的预览任务。

---

## 二、修改方案

### 2.1 TtlSchemePanel.cs

#### 2.1.1 修改预览任务判断逻辑
- **位置**：`requestPreviewVoice` 方法（第1039-1044行）
- **当前代码**：
```csharp
bool alreadyHasPreviewTask = VoiceGenerationTaskQueue?.Tasks.Any(t =>
    t.IsPreview && t.PreviewSourceName == sourceName && t.Status != VoiceGenerationTaskStatus.已完成) ?? false;
if (alreadyHasPreviewTask)
{
    return;
}
```
- **修改为**：
```csharp
bool alreadyHasPreviewTask = VoiceGenerationTaskQueue?.Tasks.Any(t =>
    t.IsPreview && t.PreviewSourceName == sourceName && t.Speed == speed && t.Status != VoiceGenerationTaskStatus.已完成) ?? false;
if (alreadyHasPreviewTask)
{
    return;
}
```
