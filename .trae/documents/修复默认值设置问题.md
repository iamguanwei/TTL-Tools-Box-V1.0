# 修复默认值设置问题

## 一、问题描述

配置项的默认值设置不正确，导致加载旧项目文件时使用了错误的默认值。

## 二、问题原因

**JSON 反序列化的行为问题：**

当使用 `DataContractJsonSerializer` 反序列化时：
- 如果字段存在于 JSON 中，使用 JSON 中的值
- 如果字段**不存在**于 JSON 中，使用**类型的默认值**（`int` 是 0，`bool` 是 false，`decimal` 是 0）

这意味着即使 `EngineProjectData` 类中定义了默认值 `= 100`，当 JSON 中没有该字段时，**会使用类型的默认值 0 而不是类中定义的默认值 100**！

**示例：**
```csharp
// EngineProjectData 类定义
public int VoicePreprocess_GlobalSpeed { get; set; } = 100;

// 但加载旧项目文件时（JSON 中没有此字段），反序列化后：
engineData.VoicePreprocess_GlobalSpeed == 0  // 类型默认值，而非 100
```

## 三、涉及文件

| 文件路径 | 修改类型 |
|----------|----------|
| `GW.TTLtoolsBox.Core\FileAccesser\ProjectFile.cs` | 修改 |

## 四、修复方案

修改 ProjectFile 中的 Get 方法，当检测到值为类型默认值（如 0）时，返回正确的业务默认值：

### 4.1 GetVoicePreprocess_GlobalSpeed 方法

```csharp
public int GetVoicePreprocess_GlobalSpeed(string engineId)
{
    if (string.IsNullOrEmpty(engineId))
    {
        return 100;
    }

    if (_projectData.EngineDataDic.TryGetValue(engineId, out EngineProjectData engineData))
    {
        // 如果值为 0（类型默认值），返回业务默认值 100
        return engineData.VoicePreprocess_GlobalSpeed == 0 ? 100 : engineData.VoicePreprocess_GlobalSpeed;
    }

    return 100;
}
```

### 4.3 GetVoicePreprocess_BlankDuration 方法

```csharp
public decimal GetVoicePreprocess_BlankDuration(string engineId)
{
    if (string.IsNullOrEmpty(engineId))
    {
        return 1;
    }

    if (_projectData.EngineDataDic.TryGetValue(engineId, out EngineProjectData engineData))
    {
        // 如果值为 0（类型默认值），返回业务默认值 1

    }

    return 1;
}
```

## 五、验证要点

1. 加载旧项目文件（不包含新字段）时，配置使用正确的默认值
2. 加载新项目文件时，配置正常读取
3. 修改配置后保存项目，重新打开项目配置应保持不变
