# 语音生成预处理配置迁移计划

## 一、需求概述

将"语音生成预处理"的以下配置项从系统配置文件(`Setting.ini`)迁移到项目文件(`.TTB`)：
- 默认朗读者
- 全局语速（值和复选框状态）
- 全局音量（值和复选框状态）
- 段间空白时长

### 默认值设置
| 配置项 | 默认值 |
|--------|--------|
| 默认朗读者 | 第一个或没有 |
| 全局语速 | 100，不勾选 |
| 全局音量 | 100，不勾选 |
| 空白时长 | 1 |

---

## 二、涉及文件

| 文件路径 | 修改类型 |
|----------|----------|
| `GW.TTLtoolsBox.Core\Entity\EngineProjectData.cs` | 新增字段 |
| `GW.TTLtoolsBox.WinFormUi\UI\Panels\VoicePreprocessPanel.cs` | 修改配置读写逻辑 |

---

## 三、实施步骤

### 步骤1：修改 EngineProjectData.cs

在 `EngineProjectData` 类中新增以下字段：

```csharp
/// <summary>
/// 获取或设置默认朗读者名称。
/// </summary>
[DataMember(Name = "voicePreprocess_defaultSpeaker")]
public string VoicePreprocess_DefaultSpeaker { get; set; } = string.Empty;

/// <summary>
/// 获取或设置全局语速值。
/// </summary>
[DataMember(Name = "voicePreprocess_globalSpeed")]
public int VoicePreprocess_GlobalSpeed { get; set; } = 100;

/// <summary>
/// 获取或设置是否启用全局语速。
/// </summary>
[DataMember(Name = "voicePreprocess_useGlobalSpeed")]
public bool VoicePreprocess_UseGlobalSpeed { get; set; } = false;

/// <summary>
/// 获取或设置全局音量值。
/// </summary>
[DataMember(Name = "voicePreprocess_globalVolume")]
public int VoicePreprocess_GlobalVolume { get; set; } = 100;

/// <summary>
/// 获取或设置是否启用全局音量。
/// </summary>
[DataMember(Name = "voicePreprocess_useGlobalVolume")]
public bool VoicePreprocess_UseGlobalVolume { get; set; } = false;

/// <summary>
/// 获取或设置段间空白时长（秒）。
/// </summary>
[DataMember(Name = "voicePreprocess_blankDuration")]
public decimal VoicePreprocess_BlankDuration { get; set; } = 1;
```

### 步骤2：修改 VoicePreprocessPanel.cs

#### 2.1 修改初始化方法 `init语音生成预处理Ui()`

将配置读取从 `Setting.GetValue()` 改为从当前引擎的项目数据读取：

| 原代码 | 新代码 |
|--------|--------|
| `Setting.GetValue(this.nud_语音生成预处理_语速设置.Name, "100")` | 从 `EngineProjectData.VoicePreprocess_GlobalSpeed` 读取 |
| `Setting.GetValue(this.nud_语音生成预处理_空白时长.Name, "1")` | 从 `EngineProjectData.VoicePreprocess_BlankDuration` 读取 |
| `Setting.GetValue(this.nud_语音生成_全局音量.Name, "100")` | 从 `EngineProjectData.VoicePreprocess_GlobalVolume` 读取 |
| `Setting.GetValue(nameof(cb_语音生成_指定全局音量), false.ToString())` | 从 `EngineProjectData.VoicePreprocess_UseGlobalVolume` 读取 |

#### 2.2 修改事件处理方法

| 方法名 | 修改内容 |
|--------|----------|
| `nud_语音生成预处理_语速设置_ValueChanged` | 保存到项目数据而非系统配置 |
| `nud_语音生成预处理_空白时长_ValueChanged` | 保存到项目数据而非系统配置 |
| `nud_语音生成_全局音量_ValueChanged` | 保存到项目数据而非系统配置 |
| `cb_语音生成_指定全局音量_CheckedChanged` | 保存到项目数据而非系统配置 |
| `cb_语音生成预处理_指定全局语速_CheckedChanged` | 保存到项目数据而非系统配置 |
| `cb_语音生成预处理_默认角色设置_SelectedIndexChanged` | 保存到项目数据而非系统配置 |

#### 2.3 新增辅助方法

新增一个方法用于获取当前引擎的项目数据，并在值变更时保存到项目数据并触发项目变更通知。

#### 2.4 处理默认朗读者初始化

修改默认朗读者的初始化逻辑：
- 从项目数据读取已保存的朗读者名称
- 如果项目数据中没有保存或保存的朗读者不存在，则选择第一个

### 步骤3：处理引擎切换

当引擎切换时，需要重新加载对应引擎的项目数据中的语音预处理配置。

---

## 四、注意事项

1. **向后兼容性**：旧版本的项目文件不包含这些字段，需要使用默认值
2. **引擎隔离**：不同引擎的配置应独立存储（当前设计已支持，每个引擎有独立的 `EngineProjectData`）
3. **项目变更通知**：配置变更后需要标记项目已修改，提示用户保存

---

## 五、验证要点

1. 新建项目时，配置使用默认值
2. 修改配置后保存项目，重新打开项目配置应保持不变
3. 切换引擎后，各引擎的配置应独立
4. 打开旧版本项目文件时，配置使用默认值
