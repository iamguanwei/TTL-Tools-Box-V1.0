# 任务重启时临时文件跳过逻辑不一致问题分析与修复计划

## 问题分析

### 问题现象
在临时文件（temp_开头）都存在的情况下，有些任务重新启动时可以快速跳过向TTL引擎提交的过程，而有些任务却每次都要重新提交。

### 根本原因

经过代码分析，发现问题出在**临时文件夹路径的同步问题**：

#### 1. 两个独立的 `_tempFolder` 实例

代码中存在两个独立的 `_tempFolder` 字段：

- **VoiceGenerationPanel._tempFolder**（[VoiceGenerationPanel.cs:303](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/UI/Panels/VoiceGenerationPanel.cs#L303)）
  - 用于**创建任务时**设置 `TempFile` 路径（[VoiceGenerationPanel.cs:1550](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/UI/Panels/VoiceGenerationPanel.cs#L1550)）

- **VoiceGenerationTaskQueue._tempFolder**（[VoiceGenerationTaskQueue.cs:909](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/Manager/VoiceGenerationTaskQueue.cs#L909)）
  - 用于**执行任务时**检查临时文件路径（[VoiceGenerationTaskQueue.cs:437](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/Manager/VoiceGenerationTaskQueue.cs#L437)）

#### 2. 路径检查逻辑

在任务执行时，会进行以下检查（[VoiceGenerationTaskQueue.cs:435-440](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/Manager/VoiceGenerationTaskQueue.cs#L435-L440)）：

```csharp
if (string.IsNullOrWhiteSpace(taskItem.TempFile) ||
    !Path.IsPathRooted(taskItem.TempFile) ||
    !taskItem.TempFile.StartsWith(_tempFolder, StringComparison.OrdinalIgnoreCase))
{
    taskItem.TempFile = Path.Combine(_tempFolder, $"temp_{task.Id}_{i + 1}.wav");
}
```

#### 3. 问题触发条件

当用户通过"设置临时目录"按钮修改临时目录时（[VoiceGenerationPanel.cs:1102-1106](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/UI/Panels/VoiceGenerationPanel.cs#L1102-L1106)）：

```csharp
if (folderBrowserDialog.ShowDialog() == DialogResult.OK)
{
    _tempFolder = folderBrowserDialog.SelectedPath;  // 只更新了 Panel 的 _tempFolder
    Setting.SetValue(nameof(临时_工作目录), _tempFolder);
}
```

**关键问题**：只更新了 `VoiceGenerationPanel._tempFolder`，**没有同步更新** `VoiceGenerationTaskQueue._tempFolder`！

#### 4. 导致的后果

| 场景 | Panel._tempFolder | Queue._tempFolder | 检查结果 | 行为 |
|------|-------------------|-------------------|----------|------|
| 从未修改临时目录 | 默认值（一致） | 默认值（一致） | 通过 | ✅ 正确跳过 |
| 修改过临时目录后 | 新路径 | 旧路径（默认值） | 失败 | ❌ 重新生成路径 |
| 修改临时目录后重启程序 | 从设置读取 | 从设置读取 | 通过 | ✅ 正确跳过 |

### 问题复现步骤

1. 启动程序（此时两个 `_tempFolder` 都是默认值）
2. 创建任务A并执行一部分（生成临时文件）
3. **修改临时目录**到新位置
4. 创建任务B并执行一部分（生成临时文件到新目录）
5. 重启任务A → ❌ 会重新提交（因为路径检查失败）
6. 重启任务B → ✅ 可能跳过（如果新目录与当前设置一致）

## 修复方案

### 方案：在修改临时目录时同步更新 TaskQueue

在 `VoiceGenerationPanel` 中修改临时目录时，同步更新 `VoiceGenerationTaskQueue` 的临时文件夹路径。

### 修改位置

**文件**：[VoiceGenerationPanel.cs](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/UI/Panels/VoiceGenerationPanel.cs)

**方法**：`bt_语音生成_设置临时目录_Click`（约第1094行）

### 修改内容

1. 在 `VoiceGenerationTaskQueue` 类中添加 `TempFolder` 属性设置器
2. 在修改临时目录时同步更新 `VoiceGenerationTaskQueue.TempFolder`

### 具体修改步骤

#### 步骤1：修改 VoiceGenerationTaskQueue.cs

添加 `TempFolder` 属性，允许外部更新临时文件夹路径：

```csharp
/// <summary>
/// 获取或设置临时文件夹路径。
/// </summary>
public string TempFolder
{
    get => _tempFolder;
    set
    {
        if (!string.IsNullOrWhiteSpace(value))
        {
            _tempFolder = value;
        }
    }
}
```

#### 步骤2：修改 VoiceGenerationPanel.cs

在设置临时目录时同步更新 TaskQueue：

```csharp
private void bt_语音生成_设置临时目录_Click(object sender, EventArgs e)
{
    using (FolderBrowserDialog folderBrowserDialog = new FolderBrowserDialog())
    {
        folderBrowserDialog.Description = "请选择临时工作目录";
        folderBrowserDialog.SelectedPath = _tempFolder;
        folderBrowserDialog.ShowNewFolderButton = true;

        if (folderBrowserDialog.ShowDialog() == DialogResult.OK)
        {
            _tempFolder = folderBrowserDialog.SelectedPath;
            Setting.SetValue(nameof(临时_工作目录), _tempFolder);
            
            // 同步更新 TaskQueue 的临时文件夹路径
            if (_voiceGenerationTaskQueue != null)
            {
                _voiceGenerationTaskQueue.TempFolder = _tempFolder;
            }
        }
    }
}
```
