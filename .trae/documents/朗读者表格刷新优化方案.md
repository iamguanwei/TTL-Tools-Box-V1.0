# 朗读者表格刷新优化方案

## 一、问题分析

### 1.1 当前刷新触发条件

| 触发时机 | 触发方法 | 刷新类型 | 是否必要 |
|---------|---------|---------|---------|
| TTL引擎连接成功 | `start连接TTL引擎()` → `bindSpeakersToGrid()` | 全表刷新 | ✅ 必要 |
| **连接成功后每60秒验证** | `handleConnectionStatusCountdown()` → `start连接TTL引擎()` | **全表刷新** | ❌ **不必要** |
| 切换TTL方案 | `cb_TTL方案_当前方案名称_SelectedIndexChanged()` | 全表刷新 | ✅ 必要 |
| 点击"重新加载"按钮 | `bt_TTL方案_重新加载朗读者_Click()` | 全表刷新 | ✅ 必要 |
| 预览声音状态变化 | `PreviewVoiceManager.StatusChanged` | 单行刷新 | ✅ 必要 |
| 单元格编辑结束 | `CellEndEdit` | 单行刷新 | ✅ 必要 |

### 1.2 问题根源

**核心问题**：连接成功后，定时器每秒检查倒计时，当倒计时归零时（60秒后），会调用 `start连接TTL引擎()` 进行"验证"，这会导致整个表格重新绑定数据。

### 1.3 影响范围

- 用户正在编辑单元格时，表格刷新会导致编辑中断
- 表格滚动位置会被重置
- 预览按钮状态可能被意外重置
- 不必要的UI刷新影响用户体验

## 二、优化方案

### 2.1 方案概述

**核心思路**：将"连接验证"与"表格刷新"解耦，只在数据真正变化时才刷新表格。

### 2.2 具体修改

#### 修改1：新增"静默验证"方法

在 `TtlSchemePanel.cs` 中新增一个静默验证方法，只验证连接状态，不刷新表格：

```csharp
/// <summary>
/// 静默验证TTL引擎连接状态（不刷新表格）。
/// </summary>
private async void verifyTtlEngineConnectionSilently()
{
    var currentEngine = TtlSchemeController?.CurrentEngineConnector;
    if (currentEngine == null)
    {
        _ttlEngineConnectionStatus = TtlEngineConnectionStatus.未连接;
        OnConnectionStatusChanged(_ttlEngineConnectionStatus, 0, 0);
        updateTtlEngineConnectionStatusLabel();
        refreshTTL方案Ui();
        return;
    }

    try
    {
        await currentEngine.ConnectAsync();
        
        _ttlEngineConnectionStatus = TtlEngineConnectionStatus.连接成功;
        _ttlEngineRetryCountdown = Constants.连接成功_验证间隔秒数;
        OnConnectionStatusChanged(_ttlEngineConnectionStatus, 0, _ttlEngineRetryCountdown);
        
        // 不刷新表格，只更新状态
    }
    catch
    {
        _ttlEngineConnectionStatus = TtlEngineConnectionStatus.连接失败;
        _ttlEngineRetryCountdown = Constants.连接失败_重试间隔秒数;
        OnConnectionStatusChanged(_ttlEngineConnectionStatus, 0, _ttlEngineRetryCountdown);
    }

    updateTtlEngineConnectionStatusLabel();
}
```

#### 修改2：修改定时器回调逻辑

修改 `handleConnectionStatusCountdown()` 方法，连接成功后的验证使用静默方法：

```csharp
case TtlEngineConnectionStatus.连接成功:
    _ttlEngineRetryCountdown--;
    if (_ttlEngineRetryCountdown <= 0)
    {
        verifyTtlEngineConnectionSilently();  // ← 改为静默验证
    }
    else
    {
        updateTtlEngineConnectionStatusLabel();
        OnConnectionStatusChanged(_ttlEngineConnectionStatus, 0, _ttlEngineRetryCountdown);
    }
    break;
```

#### 修改3：优化数据绑定方法（可选增强）

在 `bindSpeakersToGrid()` 中增加数据变化检测，避免不必要的刷新：

```csharp
/// <summary>
/// 将朗读者列表绑定到表格。
/// </summary>
/// <param name="ttl">TTL引擎连接器。</param>
/// <param name="forceRefresh">是否强制刷新。</param>
private void bindSpeakersToGrid(ITtlEngineConnector ttl, bool forceRefresh = false)
{
    var speakerArray = ttl.Speakers.Select(s => s.Clone()).ToArray();
    
    // 检查数据是否变化
    if (!forceRefresh && !hasSpeakerDataChanged(speakerArray))
    {
        return;  // 数据未变化，不刷新
    }
    
    // ... 原有绑定逻辑
}

/// <summary>
/// 检查朗读者数据是否发生变化。
/// </summary>
/// <param name="newSpeakers">新的朗读者数组。</param>
/// <returns>是否发生变化。</returns>
private bool hasSpeakerDataChanged(SpeakerInfo[] newSpeakers)
{
    var currentData = this.dgv_TTL方案_朗读者参数配置.DataSource as SpeakerInfo[];
    if (currentData == null && newSpeakers == null) return false;
    if (currentData == null || newSpeakers == null) return true;
    if (currentData.Length != newSpeakers.Length) return true;
    
    for (int i = 0; i < currentData.Length; i++)
    {
        if (currentData[i].SourceName != newSpeakers[i].SourceName ||
            currentData[i].Speed != newSpeakers[i].Speed ||
            currentData[i].Volume != newSpeakers[i].Volume ||
            currentData[i].Remark != newSpeakers[i].Remark)
        {
            return true;
        }
    }
    
    return false;
}
```

## 四、预期效果

| 场景 | 优化前 | 优化后 |
|-----|-------|-------|
| 连接成功后每60秒验证 | 全表刷新，可能中断编辑 | 仅验证连接，不刷新表格 |
| 切换TTL方案 | 全表刷新 | 全表刷新（保持不变） |
| 手动重新加载 | 全表刷新 | 全表刷新（保持不变） |
| 预览声音状态变化 | 单行刷新 | 单行刷新（保持不变） |
| 数据未变化时绑定 | 仍然刷新 | 跳过刷新 |

## 五、风险评估

| 风险 | 等级 | 应对措施 |
|-----|-----|---------|
| 静默验证失败时表格数据可能过时 | 低 | 用户可手动点击"重新加载"按钮 |
| 连接断开后表格状态不一致 | 低 | 连接失败时会更新状态标签提醒用户 |
| 代码修改引入新bug | 中 | 充分测试各种连接场景 |
