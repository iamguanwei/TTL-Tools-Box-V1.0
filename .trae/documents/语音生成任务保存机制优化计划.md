# 语音生成任务保存机制优化计划

## 一、需求概述

1. **语音生成任务保存位置变更**：将"语音生成"任务从项目文件中改为保存到系统配置中
2. **保存时机变更**：语音生成任务变化时自动保存到系统配置，不再标记项目为已修改
3. **临时文件夹检查**：确认每个任务开始时检查临时文件夹是否是最新
4. **软件单实例运行**：启动时检查是否有其他实例运行，如果有则提示并退出

## 二、现状分析

### 2.1 当前保存机制

- **保存位置**：项目文件 (`ProjectFile.SaveVoiceGenerationTasks`)
- **保存时机**：通过 `OnProjectModified()` 触发 `MainForm.markProjectModified()`，标记项目为已修改
- **触发点**（共12处）：
  - `VoiceGenerationPanel.cs:422` - 任务列表变化
  - `VoiceGenerationPanel.cs:435` - 任务列表变化
  - `VoiceGenerationPanel.cs:709` - 开始任务
  - `VoiceGenerationPanel.cs:757` - 暂停任务
  - `VoiceGenerationPanel.cs:831` - 重新启动任务
  - `VoiceGenerationPanel.cs:843` - 重新启动任务
  - `VoiceGenerationPanel.cs:1027` - 删除任务
  - `VoiceGenerationPanel.cs:1626` - 添加预览任务
  - `VoiceGenerationPanel.cs:1693` - 添加任务

### 2.2 临时文件夹检查现状

- **当前实现**：在 `VoiceGenerationTaskQueue.ExecuteTaskAsync()` 中（第444-447行）检查临时文件夹是否存在
- **问题**：仅在执行任务时检查，未在任务开始时检查临时文件夹是否是最新配置

### 2.3 单实例检查现状

- **当前实现**：无
- **入口点**：`Program.cs` 的 `Main()` 方法

```csharp
private void checkAndUpdateTempFolder()
{
    string configTempFolder = Setting.GetValue(nameof(临时_工作目录), 临时_工作目录);
    if (!string.IsNullOrEmpty(configTempFolder) && configTempFolder != _tempFolder)
    {
        _tempFolder = configTempFolder;
    }
}
```

### 步骤5：实现软件单实例运行检查

**文件**：`GW.TTLtoolsBox.WinFormUi\Program.cs`

**修改内容**：

#### 5.1 添加 Mutex 互斥检查
```csharp
[STAThread]
static void Main(string[] args)
{
    // 创建互斥体，确保只有一个实例运行
    bool createdNew;
    string mutexName = $"Global\\{System.Reflection.Assembly.GetExecutingAssembly().GetName().Name}";
    
    using (System.Threading.Mutex mutex = new System.Threading.Mutex(true, mutexName, out createdNew))
    {
        if (!createdNew)
        {
            // 已有实例运行
            MessageBox.Show(
                "程序已在运行中，请勿重复启动。",
                "提示",
                MessageBoxButtons.OK,
                MessageBoxIcon.Information);
            return;
        }
        
        Application.EnableVisualStyles();
        Application.SetCompatibleTextRenderingDefault(false);
        Application.Run(new MainForm(args));
    }
}
```

## 五、注意事项

1. **数据隔离**：不同引擎的任务数据需要分开存储（使用 engineId 作为键的一部分）
2. **错误处理**：保存和加载时需要适当的错误处理
3. **线程安全**：Setting 类的保存操作需要考虑线程安全
