# 文本拆分"忽略换行符"功能实施方案

## 一、需求概述

在"文本拆分"页面的"拆分参数"区域添加复选框"cb_文本拆分_按句子拆分_忽略换行符"，实现以下功能：
1. 默认勾选
2. 作为"按句子拆分"的子选项，联动启用/禁用
3. 配置与TTL方案关联，选中状态改变时保存
4. 选中时，先按空行拆分，每段剔除换行符后再调用原有"按句子拆分"逻辑

---

## 二、原有分割方式理解

### 2.1 原有"按句子拆分"逻辑

```
原始文本 → 按换行符(\r\n, \n, \r)分割 → 每行（含空行）独立调用句子拆分 → 合并结果
```

**关键点**：
- 每一行（包括空行）都作为独立的文本块处理
- 空行会被保留在结果中
- 换行符是"硬边界"

### 2.2 原有"按空行拆分"逻辑

```
原始文本 → 按空行分割 → 每段内移除换行符 → 合并结果
```

**关键点**：
- 空行作为分隔符
- 每段内的换行符被移除

---

## 三、功能效果分析表

### 3.1 拆分行为对比

| 场景 | 原始文本示例 | 不勾选"忽略换行符" | 勾选"忽略换行符" |
|------|-------------|-------------------|-----------------|
| 单行长文本 | `这是一段很长的文本，包含多个句子。第二句在这里。第三句。` | 正常按分隔符拆分 | 效果相同 |
| 多行无空行 | `第一行文本，<br>第二行文本。<br>第三行文本！` | 每行独立拆分，产生3段 | 合并后按分隔符拆分，产生1段（视拆分长度） |
| 多行有空行 | `第一段文本。<br><br>第二段文本！` | 空行也算一段，产生3段 | 空行分隔，每段内合并后拆分，产生2段 |
| 行内无分隔符 | `第一行无分隔符<br>第二行有句号。` | 产生2段（行作为硬边界） | 合并后产生1段 |
| 行尾有分隔符 | `第一行。<br>第二行！` | 产生2段 | 根据拆分长度确定，可能1段或2段 |
| 连续空行 | `文本A。<br><br><br>文本B！` | 每个空行算一段，产生4段 | 空行分隔后有一段是空行，产生3段 |
| 空行后有短行 | `长文本段落。<br><br>短句` | 空行也算一段，产生3段 | 空行分隔，每段独立处理，产生2段 |

### 3.2 核心差异

| 对比项 | 不勾选"忽略换行符" | 勾选"忽略换行符" |
|--------|-------------------|-----------------|
| 换行符作用 | 硬边界（每行独立处理） | 软分隔符（可被合并） |
| 空行作用 | 作为独立的一段 | 硬分隔符 |
| 处理流程 | 按换行符分割→每行独立拆分 | 按空行分割→剔除换行符→调用原有拆分 |
| 空行保留 | 保留在结果中 | 作为分隔符，不产生独立段落 |

- 添加`SetTextSplit_IgnoreLineBreaks(string engineId, bool value)`方法
- 配置键格式：`TextSplit_IgnoreLineBreaks_{engineId}`

### 步骤2：修改TextSplitHelper添加新方法 ✅（待实施）

**文件**：`GW.TTLtoolsBox.Core\TextSplit\TextSplitHelper.cs`

**操作**：
- 添加新方法`SplitTextIgnoreLineBreaks`，内部调用`SplitText`
- 方法签名：
```csharp
public static string SplitTextIgnoreLineBreaks(
    string originalText,
    int minLength,
    int maxLength,
    char[] mainSeparators,
    char[] subSeparators,
    bool trimSegments = false)
```

**实现逻辑**：
1. 先按空行拆分原始文本（复用`SplitByEmptyLine`的分割逻辑）
2. 每段剔除换行符
3. 调用原有的`SplitText`方法处理每段
4. 合并结果
