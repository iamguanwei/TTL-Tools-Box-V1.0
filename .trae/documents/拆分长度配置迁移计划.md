# "拆分长度"配置迁移计划

## 一、任务概述

将"文本拆分"面板的"拆分长度"值从项目文件（ProjectFile）迁移到系统配置（Setting）中。

### 需求要点
1. 不需要兼容旧数据，旧项目文件中的该数据直接抛弃
2. 保持与TTL方案（引擎ID）的绑定关系
3. 改变后立即保存（取消之前的项目变更状态更新机制）
4. 默认值保持100

---

## 二、涉及文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `GW.TTLtoolsBox.Core\Entity\EngineProjectData.cs` | 修改 | 移除 `TextSplit_SplitLength` 属性 |
| `GW.TTLtoolsBox.Core\FileAccesser\ProjectFile.cs` | 修改 | 移除相关方法 |
| `GW.TTLtoolsBox.WinFormUi\Helper\Setting.cs` | 修改 | 新增读写方法 |
| `GW.TTLtoolsBox.WinFormUi\UI\Panels\TextSplitPanel.cs` | 修改 | 改用Setting类存取 |

---

## 三、实施步骤

### 步骤1：修改 Setting.cs - 新增拆分长度读写方法

在 `Setting.cs` 中新增两个方法：

```csharp
/// <summary>
/// 获取文本拆分的拆分长度。
/// </summary>
/// <param name="engineId">TTL引擎ID。</param>
/// <param name="defaultValue">默认值。</param>
/// <returns>拆分长度。</returns>
public static int GetTextSplit_SplitLength(string engineId, int defaultValue = 100)

/// <summary>
/// 设置文本拆分的拆分长度。
/// </summary>
/// <param name="engineId">TTL引擎ID。</param>
/// <param name="value">拆分长度。</param>
public static void SetTextSplit_SplitLength(string engineId, int value)
```

**存储键名格式**：`TextSplit_SplitLength_{引擎ID}`

**保存机制**：设置后立即保存（`IsAutoSave = true` 时自动保存，或手动调用 `Save()`）

### 步骤2：修改 TextSplitPanel.cs - 改用Setting类

#### 2.1 修改 SaveData 方法
移除对 `projectFile.SetTextSplit_SplitLength()` 的调用。

#### 2.2 修改 LoadData 方法
将 `projectFile.GetTextSplit_SplitLength()` 改为 `Setting.GetTextSplit_SplitLength()`。

#### 2.3 修改 nud_拆分长度_ValueChanged 事件处理
移除 `onProjectModified()` 调用，改为调用 `Setting.SetTextSplit_SplitLength()` 立即保存。

### 步骤3：修改 ProjectFile.cs - 移除相关方法

移除以下方法：
- `GetTextSplit_SplitLength(string engineId, int defaultValue = 100)`
- `SetTextSplit_SplitLength(string engineId, int value)`

### 步骤4：修改 EngineProjectData.cs - 移除属性

移除 `TextSplit_SplitLength` 属性及其 `DataMember` 特性。

## 四、变更前后对比

### 存储位置
| 项目 | 变更前 | 变更后 |
|------|--------|--------|
| 存储文件 | `_默认项目_文件.json` | `Setting.ini` |
| 存储格式 | JSON（嵌套在EngineDataDic中） | INI键值对 |
| 键名格式 | `EngineDataDic[engineId].TextSplit_SplitLength` | `TextSplit_SplitLength_{engineId}` |

### 保存时机
| 项目 | 变更前 | 变更后 |
|------|--------|--------|
| 保存触发 | 值改变时触发项目修改事件 | 值改变时立即保存到Setting |
| 保存方式 | 依赖项目文件保存 | 独立立即保存 |

### 默认值
保持不变：**100**

---

## 五、注意事项

1. **不兼容旧数据**：旧项目文件中的 `TextSplit_SplitLength` 数据将被忽略，用户首次使用时使用默认值100
2. **与TTL方案绑定**：通过引擎ID区分不同方案的拆分长度设置
3. **立即保存**：值改变后立即持久化到系统配置文件
