# 任务队列恢复失败修复方案（第二版）

## 一、问题描述

软件先于TTL引擎启动时，任务处于"排队中"状态。当TTL引擎启动成功后，下方显示连接引擎成功，但任务状态依然是"排队中"，不会自动开始执行。

## 二、问题根因分析

### 2.1 调用流程分析

**场景：软件先于TTL引擎启动，任务在排队中**

```
软件启动
    └── TtlSchemePanel.InitializePanel()
            └── TtlSchemeController.StartConnectionTimer()  // 启动定时器

定时器每秒触发
    └── TtlSchemeController.ConnectionTimer_Tick()
            └── HandleFailedState()  // 引擎未启动，状态为"连接失败"
                    └── _ = ConnectAsync()  // 异步重连

引擎启动后，重连成功
    └── TtlSchemeController.ConnectAsync()
            └── _connectionStatus = 连接成功
            └── OnConnectionStatusChanged()  // 触发事件
                    └── TtlSchemePanel.TtlSchemeController_ConnectionStatusChanged()
                            └── 更新UI状态
                            └── 刷新朗读者列表
                            └── ❌ 没有调用 TryResume()！
```

### 2.2 问题本质

| 调用路径 | TryResume调用 |
|---------|--------------|
| `start连接TTL引擎()` → `ConnectAsync()` | ✅ 有调用 |
| `TtlSchemeController.HandleFailedState()` → `ConnectAsync()` | ❌ 没有调用 |

**关键问题**：
- `start连接TTL引擎()` 方法中，连接成功后会调用 `TryResume()`
- 但定时器触发的重连（`HandleFailedState()` → `ConnectAsync()`）**不会经过 `start连接TTL引擎()`**
- `TtlSchemeController_ConnectionStatusChanged` 事件处理器中**没有调用 `TryResume()`**

### 2.3 代码证据

**start连接TTL引擎() 方法（第757-766行）**：
```csharp
await TtlSchemeController.ConnectAsync();

if (TtlSchemeController.ConnectionStatus == TtlEngineConnectionStatus.连接成功)
{
    VoiceGenerationTaskQueue?.TryResume();  // ✅ 有调用
}
```

**TtlSchemeController_ConnectionStatusChanged 事件处理器（第265-293行）**：
```csharp
private void TtlSchemeController_ConnectionStatusChanged(...)
{
    // 更新UI状态
    // 刷新朗读者列表
    // ❌ 没有调用 TryResume()！
}
```

## 三、修复方案

### 方案：在事件处理器中调用 TryResume()

**修改文件**：`TtlSchemePanel.cs`

**修改位置**：`TtlSchemeController_ConnectionStatusChanged` 方法

**修改内容**：当状态变为"连接成功"时，调用 `TryResume()`

#### 修改前代码：
```csharp
private void TtlSchemeController_ConnectionStatusChanged(object sender, TtlSchemeController.ConnectionStatusChangedEventArgs e)
{
    _ttlEngineConnectionStatus = e.Status;
    _ttlEngineConnectionCountdown = e.ConnectionCountdown;
    _ttlEngineRetryCountdown = e.RetryCountdown;

    updateTtlEngineConnectionStatusLabel();
    OnConnectionStatusChanged(e.Status, e.ConnectionCountdown, e.RetryCountdown);

    bool statusChangedToConnected = e.Status == TtlEngineConnectionStatus.连接成功
        && _lastConnectionStatus != TtlEngineConnectionStatus.连接成功;
    _lastConnectionStatus = e.Status;

    if (statusChangedToConnected)
    {
        var currentEngine = TtlSchemeController?.CurrentEngineConnector;
        if (currentEngine != null)
        {
            if (currentEngine.Speakers == null || currentEngine.Speakers.Length == 0)
            {
                createSpeakersFromRoles(currentEngine, true);
            }
            else
            {
                bindSpeakersToGrid(currentEngine, true);
            }
        }
    }
}
```

#### 修改后代码：
```csharp
private void TtlSchemeController_ConnectionStatusChanged(object sender, TtlSchemeController.ConnectionStatusChangedEventArgs e)
{
    _ttlEngineConnectionStatus = e.Status;
    _ttlEngineConnectionCountdown = e.ConnectionCountdown;
    _ttlEngineRetryCountdown = e.RetryCountdown;

    updateTtlEngineConnectionStatusLabel();
    OnConnectionStatusChanged(e.Status, e.ConnectionCountdown, e.RetryCountdown);

    bool statusChangedToConnected = e.Status == TtlEngineConnectionStatus.连接成功
        && _lastConnectionStatus != TtlEngineConnectionStatus.连接成功;
    _lastConnectionStatus = e.Status;

    if (statusChangedToConnected)
    {
        var currentEngine = TtlSchemeController?.CurrentEngineConnector;
        if (currentEngine != null)
        {
            if (currentEngine.Speakers == null || currentEngine.Speakers.Length == 0)
            {
                createSpeakersFromRoles(currentEngine, true);
            }
            else
            {
                bindSpeakersToGrid(currentEngine, true);
            }
        }

        VoiceGenerationTaskQueue?.TryResume();
    }
}
```

## 四、修改清单

| 序号 | 文件 | 修改位置 | 修改内容 |
|------|------|---------|---------|
| 1 | TtlSchemePanel.cs | `TtlSchemeController_ConnectionStatusChanged()` | 添加 `TryResume()` 调用 |

## 五、修复原理

1. 无论从哪个路径触发连接成功（手动连接或定时器重连），都会触发 `ConnectionStatusChanged` 事件
2. 在事件处理器中，当状态变为"连接成功"时，调用 `TryResume()` 恢复任务队列
3. 任务队列能够正确开始执行排队中的任务

## 六、验证方法

1. 启动软件时确保TTL引擎未运行
2. 切换到"语音生成"面板，添加任务（状态为"排队中"）
3. 启动TTL引擎
4. 等待定时器自动重连成功
5. 观察任务是否自动变为"正在生成"状态
