# 实现计划：禁用"最近打开的项目"菜单项

## 需求分析

当软件启动选择"不保存"模式时，主菜单中的"新建、打开、保存、另存为"会变得不可用，现在需要让"最近打开的项目"也同样不可用。

## 当前代码分析

### 1. "不保存"模式触发位置
- **文件**: `MainForm.cs`
- **行号**: 428-436
- **逻辑**: 用户在启动对话框选择"取消"时，设置 `_isSaveOnClose = false` 并调用 `setProjectMenuEnabled(false)`

### 2. 菜单控制方法
- **文件**: `MainForm.cs`
- **行号**: 454-460
- **方法**: `setProjectMenuEnabled(bool enabled)`
- **当前控制的菜单项**:
  - `新建NToolStripMenuItem`
  - `打开OToolStripMenuItem`
  - `保存SToolStripMenuItem`
  - `另存为AToolStripMenuItem`

### 3. "最近打开的项目"菜单项
- **变量名**: `最近打开的项目HToolStripMenuItem`
- **定义位置**: `MainForm.Designer.cs` 第 347 行

## 实现步骤

### 步骤 1：修改 `setProjectMenuEnabled()` 方法

**文件**: `GW.TTLtoolsBox.WinFormUi\MainForm.cs`

**修改内容**: 在 `setProjectMenuEnabled()` 方法中添加对 `最近打开的项目HToolStripMenuItem` 的控制

**修改前** (第 454-460 行):
```csharp
private void setProjectMenuEnabled(bool enabled)
{
    this.新建NToolStripMenuItem.Enabled = enabled;
    this.打开OToolStripMenuItem.Enabled = enabled;
    this.保存SToolStripMenuItem.Enabled = enabled;
    this.另存为AToolStripMenuItem.Enabled = enabled;
}
```

**修改后**:
```csharp
private void setProjectMenuEnabled(bool enabled)
{
    this.新建NToolStripMenuItem.Enabled = enabled;
    this.打开OToolStripMenuItem.Enabled = enabled;
    this.最近打开的项目HToolStripMenuItem.Enabled = enabled;
    this.保存SToolStripMenuItem.Enabled = enabled;
    this.另存为AToolStripMenuItem.Enabled = enabled;
}
```

## 影响范围

此修改会影响所有调用 `setProjectMenuEnabled()` 的场景：

| 场景 | 调用方式 | 影响 |
|------|----------|------|
| 启动时选择"取消" | `setProjectMenuEnabled(false)` | "最近打开的项目"将被禁用 ✓ |
| 加载项目成功 | `setProjectMenuEnabled(true)` | "最近打开的项目"将被启用 |
| 另存为成功 | `setProjectMenuEnabled(true)` | "最近打开的项目"将被启用 |
| 新建项目后 | `setProjectMenuEnabled(true)` | "最近打开的项目"将被启用 |
| 打开项目成功 | `setProjectMenuEnabled(true)` | "最近打开的项目"将被启用 |
| 从最近项目打开后 | `setProjectMenuEnabled(true)` | "最近打开的项目"将被启用 |

## 验证方法

1. 启动软件，在对话框中选择"取消"
2. 确认主菜单中以下菜单项均不可用：
   - 新建(&N)...
   - 打开(&O)...
   - 最近打开的项目(&H)...
   - 保存(&S)
   - 另存为(&A)...
