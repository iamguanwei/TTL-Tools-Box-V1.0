# 音频播放器文件释放问题修复计划

## 问题分析

### 问题现象
调用内部播放器播放音频后，操作文件时提示"正在使用中"。

### 实际使用的播放器确认

经过完整调用链追踪，确认实际使用的是 **`AudioPlayer`**（使用 `System.Windows.Media.MediaPlayer`），而非 `AudioService`：

```
VoiceGenerationPanel.PlaySound?.Invoke(filePath)
    ↓
MainForm.playSound(filePath)  [MainForm.cs:165]
    ↓
_audioPlayer.Play(filePath)   [MainForm.cs:167]
    ↓
AudioPlayer (使用 System.Windows.Media.MediaPlayer)  [AudioPlayer.cs]
```

**注意**：`AudioService`（使用 `SoundPlayer`）虽然存在于代码中，但从未被实例化使用。

### 问题根因
在 [AudioPlayer.cs](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/Helper/AudioPlayer.cs) 中：

1. **文件锁定机制**：`MediaPlayer.Open()` 方法会锁定音频文件
2. **缺少自动释放**：播放完成后没有监听 `MediaEnded` 事件来释放文件
3. **资源未及时释放**：只有在手动调用 `Stop()` 或播放新文件时才会释放旧文件

### 问题代码位置

[AudioPlayer.cs:48-74](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/Helper/AudioPlayer.cs#L48-L74)：

```csharp
public void Play(string filePath)
{
    if (string.IsNullOrWhiteSpace(filePath) || !File.Exists(filePath))
    {
        return;
    }

    Stop();

    try
    {
        _mediaPlayer = new System.Windows.Media.MediaPlayer();
        _mediaPlayer.MediaOpened += (s, e) =>
        {
            if (_mediaPlayer != null)
            {
                _mediaPlayer.Play();
            }
        };
        _mediaPlayer.Open(new Uri(filePath));  // 这里锁定文件，但播放完成后没有释放
    }
    catch
    {
        _mediaPlayer?.Close();
        _mediaPlayer = null;
    }
}
```

## 修复方案

### 修改内容
在 `AudioPlayer.cs` 的 `Play()` 方法中添加 `MediaEnded` 事件处理，播放完成后自动释放文件资源。

### 具体修改步骤

修改 `AudioPlayer.cs` 第 57-74 行，添加 `MediaEnded` 事件：

```csharp
public void Play(string filePath)
{
    if (string.IsNullOrWhiteSpace(filePath) || !File.Exists(filePath))
    {
        return;
    }

    Stop();

    try
    {
        _mediaPlayer = new System.Windows.Media.MediaPlayer();
        _mediaPlayer.MediaOpened += (s, e) =>
        {
            if (_mediaPlayer != null)
            {
                _mediaPlayer.Play();
            }
        };
        _mediaPlayer.MediaEnded += (s, e) =>
        {
            Stop();  // 播放完成后自动释放资源
        };
        _mediaPlayer.Open(new Uri(filePath));
    }
    catch
    {
        _mediaPlayer?.Close();
        _mediaPlayer = null;
    }
}
```

## 影响范围

- 仅修改 `AudioPlayer.cs` 文件
- 不影响其他模块
- 不改变现有功能行为，只是添加了自动释放机制

## 验证方法

1. 播放一个音频文件
2. 等待播放完成
3. 尝试删除或移动该音频文件
4. 应该可以正常操作，不再提示"正在使用中"
