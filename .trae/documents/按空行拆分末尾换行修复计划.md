# 按空行拆分末尾换行处理修复计划

## 问题描述
当原始文本以多个`\r\n`结尾时，拆分后的最终文本会丢失部分换行。

## 用户期望行为
| 输入 | 期望输出 |
|------|----------|
| "原始文本\r\n" | "原始文本" |
| "原始文本\r\n\r\n" | "原始文本\r\n" |
| "原始文本\r\n\r\n\r\n" | "原始文本\r\n\r\n" |

规则：末尾换行数减1（最小为0）

## 当前代码问题
当前代码在处理末尾空行时，只追加1个`\r\n`：
```csharp
if (_emptyLineCount > 0)
{
    _currentParagraph.Append("\r\n");
}
```

应该改为追加 `(空行数-1)` 个换行。

## 修复方案

**文件**: `GW.TTLtoolsBox.Core\TextSplit\TextSplitHelper.cs`

将末尾处理逻辑从：
```csharp
if (_currentParagraph.Length > 0)
{
    if (_emptyLineCount > 0)
    {
        _currentParagraph.Append("\r\n");
    }
    ...
}
```

修改为：
```csharp
if (_currentParagraph.Length > 0)
{
    for (int i = 0; i < _emptyLineCount - 1; i++)
    {
        _currentParagraph.Append("\r\n");
    }
    ...
}
```

## 验证用例
- "原始文本\r\n" → 空行数=1 → 追加0个 → "原始文本" ✓
- "原始文本\r\n\r\n" → 空行数=2 → 追加1个 → "原始文本\r\n" ✓
- "原始文本\r\n\r\n\r\n" → 空行数=3 → 追加2个 → "原始文本\r\n\r\n" ✓
