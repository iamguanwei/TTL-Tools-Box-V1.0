# 多音字替换"不替换"选项记录问题修复计划

## 一、问题分析

### 1.1 用户描述的问题

选择"不替换"后，在按"上一条"返回之前选过的项，结果"不替换"那个自己变成了另外一个选项。

### 1.2 问题根源

在 `ReplaceItem.UpdateReplaceText` 方法中：

```csharp
public void UpdateReplaceText(string newReplaceText)
{
    if (newReplaceText != null)  // 问题：当传入 null 时，不会执行下面的代码
    {
        var ps = getSelectedPolyphonicScheme();
        var index = Array.FindIndex(ps.ReplaceStrings, s => s.Equals(newReplaceText) == true);
        if (index >= 0) _selectedReplaceStringIndex = index;
        else _selectedReplaceStringIndex = -1;

        IsReplaceTextUpdated = true;
    }
}
```

当用户选择"不替换"时，`SetSelectionIndex(item, 0)` 会调用 `item.UpdateReplaceText(null)`，但由于 `null != null` 为 `false`，`IsReplaceTextUpdated` 不会被设置为 `true`。

这导致：
1. 用户选择"不替换"后，`IsReplaceTextUpdated` 仍然是 `false`
2. 当用户按"上一条"返回时，`CalculateSelectedIndex` 方法检查 `IsReplaceTextUpdated`，由于是 `false`，它会根据其他逻辑决定选择，而不是使用用户之前选择的"不替换"

---

## 二、修改方案

### 2.1 修改 `ReplaceItem.UpdateReplaceText` 方法

**文件**：`GW.TTLtoolsBox.Core\PolyReplace\ReplaceItem.cs`

**修改内容**：允许 `null` 值表示"不替换"，并正确设置 `IsReplaceTextUpdated`

**修改后的代码**：

```csharp
/// <summary>
/// 更新替换的文本。
/// </summary>
/// <param name="newReplaceText">新的替换文本，null表示选择"不替换"。</param>
public void UpdateReplaceText(string newReplaceText)
{
    if (newReplaceText == null)
    {
        _selectedReplaceStringIndex = -1;
        IsReplaceTextUpdated = true;
    }
    else
    {
        var ps = getSelectedPolyphonicScheme();
        var index = Array.FindIndex(ps.ReplaceStrings, s => s.Equals(newReplaceText) == true);
        if (index >= 0) _selectedReplaceStringIndex = index;
        else _selectedReplaceStringIndex = -1;

        IsReplaceTextUpdated = true;
    }
}
```

或者简化为：

```csharp
/// <summary>
/// 更新替换的文本。
/// </summary>
/// <param name="newReplaceText">新的替换文本，null表示选择"不替换"。</param>
public void UpdateReplaceText(string newReplaceText)
{
    if (newReplaceText == null)
    {
        _selectedReplaceStringIndex = -1;
    }
    else
    {
        var ps = getSelectedPolyphonicScheme();
        var index = Array.FindIndex(ps.ReplaceStrings, s => s.Equals(newReplaceText) == true);
        if (index >= 0) _selectedReplaceStringIndex = index;
        else _selectedReplaceStringIndex = -1;
    }

    IsReplaceTextUpdated = true;
}
```

## 四、测试场景

修改完成后，需要测试以下场景：

1. **选择"不替换"后返回**：选择"不替换"，按"下一条"，再按"上一条"返回，验证仍然显示"不替换"被选中

2. **选择其他选项后返回**：选择某个替换选项，按"下一条"，再按"上一条"返回，验证仍然显示之前选择的选项

3. **组内传递"不替换"**：首次进入组时选择"不替换"，验证后续同组项也默认选择"不替换"

4. **组完成后修改**：组完成后通过"上一条"返回修改选择，验证仅修改当前项
