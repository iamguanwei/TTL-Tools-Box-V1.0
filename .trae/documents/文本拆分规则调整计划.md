# 文本拆分规则调整计划

## 需求分析

### 1. 按句子拆分规则调整
- 如果某一段**只包含**"整句分割符号"（`mainSeparators`中的字符，其一或多个组合）或"半句分割符号"（`subSeparators`中的字符，其一或多个组合），则将其与上一段合并
- 如果没有上一段，则舍弃该段

### 2. 按对话拆分规则调整
- 如果某一段**只包含**"对话分割符号"（`mainSeparators`中的字符，其一或多个组合），则将其与上一段合并
- 如果没有上一段，则舍弃该段

## 实现方案

### 方案概述
创建一个通用的私有方法 `mergeSeparatorOnlySegments`，用于处理拆分结果中仅包含分隔符的段落合并逻辑，避免代码重复。

### 具体步骤

#### 步骤1：新增私有方法 `mergeSeparatorOnlySegments`
- **位置**：`#region private` 区域
- **功能**：检查拆分结果中是否有段落仅由指定分隔符组成，若有则合并到上一段或舍弃
- **参数**：
  - `List<string> segments`：拆分后的段落列表
  - `HashSet<char> separators`：需要检查的分隔符集合
- **逻辑**：
  1. 遍历段落列表
  2. 判断当前段落是否所有字符都在分隔符集合中
  3. 如果是且存在上一段 → 合并到上一段
  4. 如果是且不存在上一段 → 舍弃（不添加到结果）
  5. 否则 → 保留该段

#### 步骤2：修改 `SplitText`（按句子拆分）方法
- **位置**：第一个重载方法，约第26-60行
- **修改点**：在步骤4（拼接结果）之前，调用 `mergeSeparatorOnlySegments` 进行后处理
- **处理逻辑**：
  - 先用 `mainSeparators` 调用一次合并方法
  - 再用 `subSeparators` 调用一次合并方法
  - 确保两种分隔符的情况都能被处理

#### 步骤3：修改 `SplitText`（按对话拆分）方法
- **位置**：第二个重载方法，约第70-105行
- **修改点**：在步骤5（拼接结果）之前，调用 `mergeSeparatorOnlySegments` 进行后处理
- **处理逻辑**：
  - 使用所有对话分隔符（开始和结束符号）构建分隔符集合
  - 调用合并方法处理

## 示例说明

### 按句子拆分示例
假设 `mainSeparators = ['。', '！', '？']`，`subSeparators = ['，', '；']`

**输入**：
```
你好。！！！今天天气不错。
```

**当前输出**：
```
你好。
！！！
今天天气不错。
```

**调整后输出**：
```
你好。！！！
今天天气不错。
```

### 按对话拆分示例
假设 `mainSeparators = ['"', '"', '「', '」']`

**输入**：
```
他说：「」然后离开了。
```

**当前输出**：
```
他说：
「」
然后离开了。
```

**调整后输出**：
```
他说：「」
然后离开了。
```
