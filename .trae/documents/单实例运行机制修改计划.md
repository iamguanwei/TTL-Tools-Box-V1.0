# 单实例运行机制修改计划

## 一、问题描述

当前实现使用程序集名称作为 Mutex 标识：

```csharp
string mutexName = $"Global\\{Assembly.GetExecutingAssembly().GetName().Name}";
```

这会导致：
- 所有同名程序集共享同一个 Mutex
- 从不同路径启动的相同 exe 会互相阻止
- **不满足需求**：仅限于同一 exe 文件

## 二、修改目标

使单实例检查仅针对**同一 exe 文件路径**：
- `C:\文件夹A\TTL Tools Box.exe` 和 `C:\文件夹B\TTL Tools Box.exe` 可以同时运行
- 同一 exe 文件的重复启动会被阻止

## 三、修改方案

### 文件：`Program.cs`

**修改前**：
```csharp
string mutexName = $"Global\\{Assembly.GetExecutingAssembly().GetName().Name}";
```

**修改后**：
```csharp
string exePath = Application.ExecutablePath;
string mutexKey = Convert.ToBase64String(System.Text.Encoding.UTF8.GetBytes(exePath))
    .Replace('/', '_')
    .Replace('+', '-')
    .Replace('=', '_');
string mutexName = $"Global\\TTLToolsBox_{mutexKey}";
```

### 方案说明

1. **获取 exe 完整路径**：`Application.ExecutablePath` 返回当前运行的 exe 文件完整路径
2. **转换为合法 Mutex 名称**：
   - 将路径进行 Base64 编码
   - 替换 Mutex 名称中的非法字符（`/`、`+`、`=`）
3. **添加前缀**：使用 `TTLToolsBox_` 前缀，避免与其他全局 Mutex 冲突

### 示例

| exe 路径 | Mutex 名称 |
|---------|-----------|
| `C:\A\TTL Tools Box.exe` | `Global\TTLToolsBox_QzpcQVxUVEwgVG9vbHMgQm94LmV4ZQ__` |
| `D:\B\TTL Tools Box.exe` | `Global\TTLToolsBox_RDpcQlxUVEwgVG9vbHMgQm94LmV4ZQ__` |

两个不同路径产生不同的 Mutex 名称，因此可以同时运行。

## 四、影响范围

- 仅修改 `Program.cs` 文件
- 不影响其他功能
- 修改后需要重新编译

## 五、测试要点

1. 从路径 A 启动程序，再次从路径 A 启动 → 应提示"程序已在运行中"
2. 从路径 A 启动程序，从路径 B 启动 → 应正常启动，两个实例同时运行
