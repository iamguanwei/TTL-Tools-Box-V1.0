# 文本拆分面板"拆分长度"纳入项目文件保存 - 实施计划

## 需求分析

将"文本拆分"面板的"拆分长度"数值从系统配置（INI文件）迁移到项目文件保存，与TTL方案绑定，默认值设为100。

### 当前状态
- 拆分长度通过 `Setting.SetValue/GetValue` 保存到 `Setting.ini` 文件
- 与TTL方案无关，属于全局用户偏好设置

### 目标状态
- 拆分长度保存到项目文件（JSON格式）
- 与TTL方案绑定，存储在 `EngineProjectData` 中
- 默认值从10改为100

## 实施步骤

### 步骤1：修改数据实体类

**文件**: `GW.TTLtoolsBox.Core\Entity\EngineProjectData.cs`

添加拆分长度属性：
```csharp
/// <summary>
/// 获取或设置文本拆分的拆分长度。
/// </summary>
[DataMember(Name = "textSplit_splitLength")]
public int TextSplit_SplitLength { get; set; } = 100;
```

### 步骤2：修改项目文件管理类

**文件**: `GW.TTLtoolsBox.Core\FileAccesser\ProjectFile.cs`

添加获取和设置拆分长度的方法：
```csharp
/// <summary>
/// 获取文本拆分的拆分长度。
/// </summary>
/// <param name="engineId">TTL引擎ID</param>
/// <param name="defaultValue">默认值</param>
/// <returns>拆分长度</returns>
public int GetTextSplit_SplitLength(string engineId, int defaultValue = 100)

/// <summary>
/// 设置文本拆分的拆分长度。
/// </summary>
/// <param name="engineId">TTL引擎ID</param>
/// <param name="value">拆分长度</param>
public void SetTextSplit_SplitLength(string engineId, int value)
```

### 步骤3：修改文本拆分面板

**文件**: `GW.TTLtoolsBox.WinFormUi\UI\Panels\TextSplitPanel.cs`

1. 添加 `CurrentEngineId` 属性，用于获取当前引擎ID
2. 修改 `SaveData` 方法，增加保存拆分长度
3. 修改 `LoadData` 方法，增加加载拆分长度
4. 修改 `init文本拆分Ui` 方法：
   - 移除从Setting读取拆分长度的逻辑
   - 默认值设为100
5. 修改 `bt_开始拆分_Click` 方法：
   - 移除通过Setting保存拆分长度的逻辑

### 步骤4：修改主窗体

**文件**: `GW.TTLtoolsBox.WinFormUi\MainForm.cs`

1. 修改 `save文本拆分PanelData` 方法，传递当前引擎ID
2. 修改 `load文本拆分PanelData` 方法，传递当前引擎ID
3. 在引擎切换事件处理中，确保正确保存和加载文本拆分面板数据

## 数据流程

### 保存流程
1. 用户修改拆分长度 → 点击"开始拆分"按钮
2. `MainForm` 调用 `save文本拆分PanelData()`
3. `TextSplitPanel.SaveData()` 保存原始文本、最终文本和拆分长度到 `ProjectFile`
4. `ProjectFile.SetTextSplit_SplitLength(engineId, value)` 保存到对应引擎的数据

### 加载流程
1. 用户切换TTL方案或打开项目
2. `MainForm` 调用 `load文本拆分PanelData()`
3. `TextSplitPanel.LoadData()` 从 `ProjectFile` 加载数据
4. `ProjectFile.GetTextSplit_SplitLength(engineId, 100)` 获取对应引擎的拆分长度

## 注意事项

1. **向后兼容**：旧项目文件没有拆分长度字段时，使用默认值100
2. **引擎绑定**：拆分长度存储在 `EngineProjectData` 中，每个TTL方案可以有独立的拆分长度设置
3. **废止系统配置**：移除通过 `Setting` 类保存/读取拆分长度的代码
