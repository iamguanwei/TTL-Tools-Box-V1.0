# 预览任务未等待引擎连接成功的问题分析

## 问题描述

用户提交预览任务后，任务没有等待引擎连接成功就自己变成"正在生成"状态。

## 问题根源

在 `VoiceGenerationTaskQueue.cs` 的 `ProcessNextTask()` 方法中，连接状态检查逻辑有问题：

```csharp
// 当前代码（第348-350行）
var connectionStatus = _getConnectionStatus?.Invoke() ?? TtlEngineConnectionStatus.未连接;
if (connectionStatus == TtlEngineConnectionStatus.未连接 ||
    connectionStatus == TtlEngineConnectionStatus.连接失败)
{
    // 等待连接...
}
```

**问题**：这段代码只排除了 `未连接` 和 `连接失败`，但**没有排除 `连接中`**！

当连接状态是 `连接中` 时，任务会直接跳过检查，进入 "正在生成" 状态，但实际上引擎还没有连接成功。

## 调用链分析

1. TtlSchemePanel 提交预览任务时调用：
   ```csharp
   VoiceGenerationTaskQueue.Start();  // 直接调用 Start()
   ```

2. `Start()` 方法直接调用 `ProcessNextTask()`，没有检查连接状态：
   ```csharp
   public void Start()
   {
       lock (_lock)
       {
           if (_isRunning) return;
       }
       ProcessNextTask();  // 直接调用，无连接状态检查
   }
   ```

3. `ProcessNextTask()` 中的连接状态检查不完整

## 修复方案

修改 `ProcessNextTask()` 中的连接状态检查逻辑，改为**只允许 `连接成功` 状态**：

```csharp
// 修改前
if (connectionStatus == TtlEngineConnectionStatus.未连接 ||
    connectionStatus == TtlEngineConnectionStatus.连接失败)

// 修改后
if (connectionStatus != TtlEngineConnectionStatus.连接成功)
```

这样当状态是 `未连接`、`连接中`、`连接失败` 时，都会等待连接。

### 步骤1：修改 VoiceGenerationTaskQueue.cs

修改 `ProcessNextTask()` 方法中的连接状态检查条件（约第348-350行）。
