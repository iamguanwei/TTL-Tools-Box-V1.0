# 语音生成表格右键菜单优化方案

## 一、问题描述

语音生成面板的任务清单表格（`dgv_语音生成_任务清单`）在数据行很少时，表格下方会有空白区域。当前实现在空白区域右击也会弹出菜单，用户希望**只有在数据行上右击才弹出菜单**。

## 二、问题原因

当前实现方式：
- DataGridView 的 `ContextMenuStrip` 属性直接绑定了 `cms_语音生成_任务控制`
- 这导致在表格的任何位置右击都会触发菜单弹出

## 三、解决方案

在 `cms_语音生成_任务控制_Opening` 事件处理方法中，添加判断逻辑：
1. 获取鼠标在表格中的点击位置
2. 使用 `HitTest` 方法检测点击位置是否在数据行上
3. 如果不在数据行上，设置 `e.Cancel = true` 取消菜单显示

### 修改内容

在方法开头添加以下代码：

```csharp
// 获取鼠标在表格中的位置
Point mousePosition = dgv_语音生成_任务清单.PointToClient(Cursor.Position);
DataGridView.HitTestInfo hitTest = dgv_语音生成_任务清单.HitTest(mousePosition.X, mousePosition.Y);

// 如果不在数据行上，取消菜单显示
if (hitTest.Type != DataGridViewHitTestType.Cell || hitTest.RowIndex < 0)
{
    e.Cancel = true;
    return;
}
```

### 完整修改后的方法结构

```csharp
private void cms_语音生成_任务控制_Opening(object sender, System.ComponentModel.CancelEventArgs e)
{
    // 获取鼠标在表格中的位置
    Point mousePosition = dgv_语音生成_任务清单.PointToClient(Cursor.Position);
    DataGridView.HitTestInfo hitTest = dgv_语音生成_任务清单.HitTest(mousePosition.X, mousePosition.Y);

    // 如果不在数据行上，取消菜单显示
    if (hitTest.Type != DataGridViewHitTestType.Cell || hitTest.RowIndex < 0)
    {
        e.Cancel = true;
        return;
    }

    // 原有逻辑保持不变...
    bool hasSelectedRows = dgv_语音生成_任务清单.SelectedRows.Count > 0;
    // ...
}
```

## 五、影响范围

- 仅修改 `VoiceGenerationPanel.cs` 文件

- 不影响其他功能，原有菜单项的启用/禁用逻辑保持不变

## 六、测试要点

1. 表格有数据时，在数据行上右击 → 菜单正常弹出
2. 表格有数据时，在空白区域右击 → 菜单不弹出
3. 表格无数据时，右击 → 菜单不弹出
4. 菜单项的启用/禁用状态正常工作
