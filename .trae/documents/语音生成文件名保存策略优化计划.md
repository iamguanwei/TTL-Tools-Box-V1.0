# 语音生成文件名保存策略优化计划

## 一、问题分析

### 当前问题
当用户输入的文件名只包含数字（如 `1.wav`）且只有一个任务时：
1. `SplitTrailingNumber("1")` 返回 `["", "1"]`
2. `mFileName` 被设置为空字符串 `""`
3. 单个任务时 `fileNameSuffix` 为空
4. 最终文件名 = `""` + `""` + `".wav"` = `.wav`（丢失了主文件名）

### 当前代码逻辑（第1407-1421行）
```csharp
int fileCount = 1;
{
    var strArray = FileHelper.SplitTrailingNumber(mFileName);
    if (strArray != null && strArray.Length > 0)
    {
        mFileName = strArray[0];  // 问题：这里会修改原始文件名
        if (strArray.Length > 1) fileCount = int.Parse(strArray[1]);
    }
}

bool isSingleTask = paragraphArray.Length == 1;
foreach (var parapraph in paragraphArray)
{
    string fileNameSuffix = isSingleTask ? string.Empty : FileHelper.PadNumber(fileCount++, fileCountMax);
    task.SaveFile = Path.Combine(folderPath, $"{mFileName}{fileNameSuffix}{eFileName}");
}
```

## 二、需求明确

### 单个任务
- 输入什么文件名就保存成什么文件名
- 不处理末尾数字，直接使用原始文件名

### 多个任务
**情况A：文件名末尾有数字（如 `test01`）**
- 第一个任务：`test01`（使用原始数字）
- 第二个任务：`test02`（累加）
- 第三个任务：`test03`（累加）

**情况B：文件名末尾没有数字（如 `test`）**
- 第一个任务：`test`（不加数字）
- 第二个任务：`test02`（从02开始，因为第一个任务占用了"无数字"位置）
- 第三个任务：`test03`（累加）

### 保留机制
- 根据任务数量自动计算数字位数（如10个任务用2位，100个任务用3位）

    int fileCount = 1;
    string baseFileName = mFileName;
    bool hasTrailingNumber = false;

    var strArray = FileHelper.SplitTrailingNumber(mFileName);
    if (strArray != null && strArray.Length > 1)
    {
        baseFileName = strArray[0];
        fileCount = int.Parse(strArray[1]);

        {
            fileNameSuffix = FileHelper.PadNumber(fileCount++, fileCountMax);
        }

        task.SaveFile = Path.Combine(folderPath, $"{baseFileName}{fileNameSuffix}{eFileName}");
        task.Speed = speed;
        task.SpaceTime = spaceTime;

        if (cb_语音生成_自动开始任务.Checked)
        {
            task.Status = VoiceGenerationTaskStatus.排队中;
        }

        List<VoiceGenerationTaskItem> taskItems = new List<VoiceGenerationTaskItem>();

        // ... 后续代码保持不变

        taskIndex++;
    }
}
