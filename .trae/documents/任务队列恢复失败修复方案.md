# 任务队列恢复失败修复方案

## 一、问题描述

软件先于TTL引擎启动时，任务处于"排队中"状态。当TTL引擎启动成功后，下方显示连接引擎成功，但任务状态依然是"排队中"，不会自动开始执行。需要手动停止后再启动，才会变成"正在生成"状态。

## 二、问题根因分析

### 2.1 状态获取存在时序问题

**调用链路**：
```
start连接TTL引擎()
    └── await TtlSchemeController.ConnectAsync()
            └── TtlSchemeController._connectionStatus = 连接成功
            └── 触发 ConnectionStatusChanged 事件（异步）
    └── if (TtlSchemeController.ConnectionStatus == 连接成功)  ✓
            └── VoiceGenerationTaskQueue?.TryResume()
                    └── _getConnectionStatus?.Invoke()
                            └── _ttlSchemePanel.ConnectionStatus
                                    └── _ttlEngineConnectionStatus  ✗ 此时仍是旧值！
    └── TtlSchemeController_ConnectionStatusChanged 事件处理器执行
            └── _ttlEngineConnectionStatus = e.Status  // 更新状态（晚了）
```

### 2.2 问题本质

| 组件 | 状态来源 | 更新时机 |
|------|---------|---------|
| TtlSchemeController | `_connectionStatus` 字段 | ConnectAsync() 完成时立即更新 |
| TtlSchemePanel | `_ttlEngineConnectionStatus` 字段 | 事件处理器执行时更新 |
| VoiceGenerationTaskQueue | 通过委托获取 `_ttlSchemePanel.ConnectionStatus` | 获取的是Panel的字段 |

**时序问题**：`TryResume()` 调用时，Panel 的 `_ttlEngineConnectionStatus` 字段尚未更新，导致 `TryResume()` 获取到的状态不是"连接成功"，提前返回。

### 2.3 关键代码

**MainForm.cs 第1168行**：
```csharp
_voiceGenerationPanel.GetEngineConnectionStatus = () => _ttlSchemePanel?.ConnectionStatus ?? TtlEngineConnectionStatus.未连接;
```

**TtlSchemePanel.cs 第198行**：
```csharp
public TtlEngineConnectionStatus ConnectionStatus => _ttlEngineConnectionStatus;
```

**VoiceGenerationTaskQueue.cs 第289-290行**：
```csharp
var connectionStatus = _getConnectionStatus?.Invoke() ?? TtlEngineConnectionStatus.未连接;
if (connectionStatus != TtlEngineConnectionStatus.连接成功) return;  // 提前返回！
```

## 三、修复方案

### 方案：修改委托指向TtlSchemeController

**修改文件**：`MainForm.cs`

**修改位置**：`initVoiceGenerationPanel()` 方法中设置委托的位置

**修改内容**：将状态获取委托从 `TtlSchemePanel.ConnectionStatus` 改为 `TtlSchemeController.ConnectionStatus`

#### 修改前代码：
```csharp
_voiceGenerationPanel.GetEngineConnectionStatus = () => _ttlSchemePanel?.ConnectionStatus ?? TtlEngineConnectionStatus.未连接;
```

#### 修改后代码：
```csharp
_voiceGenerationPanel.GetEngineConnectionStatus = () => _ttlSchemeController?.ConnectionStatus ?? TtlEngineConnectionStatus.未连接;
```

## 四、修复原理

1. `TtlSchemeController.ConnectionStatus` 在 `ConnectAsync()` 完成时立即更新
2. `TryResume()` 调用时，Controller 的状态已经是"连接成功"
3. 任务队列能够正确获取到最新状态，开始执行任务

## 五、实施步骤

1. 修改 `MainForm.cs` 中的 `initVoiceGenerationPanel()` 方法
2. 将 `_voiceGenerationPanel.GetEngineConnectionStatus` 委托改为从 `_ttlSchemeController` 获取状态

## 六、验证方法

1. 启动软件时确保TTL引擎未运行
2. 切换到"语音生成"面板，添加任务（状态为"排队中"）
3. 启动TTL引擎
4. 观察连接成功后，任务是否自动变为"正在生成"状态
