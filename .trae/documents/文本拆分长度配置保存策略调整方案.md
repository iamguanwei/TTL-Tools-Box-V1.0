# 文本拆分"拆分长度"配置保存策略调整方案

## 一、需求概述

调整"文本拆分"面板中"拆分长度"值的保存策略：

1. 系统配置中只保存最后一次修改过的值
2. 每个项目文件也要保存该值
3. 读取项目文件时，优先从项目文件加载，失败则从系统配置加载，再失败则使用默认值
4. 软件启动时选择"是/否/取消"，都从系统配置中加载该值；主菜单"新建"时保持UI现有值不变
5. 从项目文件加载的值不应保存到系统配置，除非用户又调整过该值

### 2.3 现有加载流程

**软件启动时** (`MainForm_Shown` 方法):

1. 如果有命令行传入的项目文件路径，加载该项目文件
2. 否则弹出对话框询问用户：

   * 选择"是"：加载默认项目文件

   * 选择"否"：清空内容，从系统配置加载

   * 选择"取消"：清空内容，不保存

**新建项目时** (`新建NToolStripMenuItem_Click` 方法):

* 清空项目内容，使用默认项目文件路径

**加载项目文件时** (`loadProjectFile` 方法):

**TextSplitPanel.LoadData** 方法:

* 当前只从系统配置加载拆分长度：`Setting.GetTextSplit_SplitLength(CurrentEngineId, 100)`

### 3.2 TextSplitPanel.cs 修改

#### 3.2.1 添加字段

/// <summary>
/// 拆分长度值是否来自项目文件。
/// 用于判断是否需要在UI修改时保存到系统配置。
/// </summary>
private bool _isSplitLengthFromProject = false;

#### 3.2.2 修改 LoadData 方法

```csharp
/// <summary>
/// 从项目文件加载面板数据。
/// </summary>
/// <param name="projectFile">项目文件实例。</param>
/// <param name="loadFromSystemConfig">是否从系统配置加载拆分长度（默认为false）。</param>
/// <param name="keepSplitLength">是否保持UI现有拆分长度值不变（默认为false）。</param>
public void LoadData(ProjectFile projectFile, bool loadFromSystemConfig = false, bool keepSplitLength = false)
{
    if (projectFile != null)
    {
        _isLoading = true;
        UpdateUi(() =>
        {
            this.tb_原始文本.Text = projectFile.TextSplit_OriginalText;
            this.tb_最终文本.Text = projectFile.TextSplit_FinalText;
            
            // 如果需要保持UI现有值，则不修改拆分长度
            if (!keepSplitLength)
            {
                // 加载拆分长度：优先级为 项目文件 > 系统配置 > 默认值
                int splitLength = 100; // 默认值
                _isSplitLengthFromProject = false;
                
                if (!loadFromSystemConfig && projectFile.TextSplit_SplitLength.HasValue)
                {
                    // 从项目文件加载
                    splitLength = projectFile.TextSplit_SplitLength.Value;
                    _isSplitLengthFromProject = true;
                }
                else
                {
                    // 从系统配置加载
                    splitLength = Setting.GetTextSplit_SplitLength(CurrentEngineId, 100);
                    _isSplitLengthFromProject = false;
                }
                
                this.nud_拆分长度.Value = splitLength;
            }
            
            this.cb_文本拆分_按句子拆分_忽略换行符.Checked = Setting.GetTextSplit_IgnoreLineBreaks(CurrentEngineId, true);
        });
        _isLoading = false;
    }
}
```

#### 3.2.3 修改 SaveData 方法

```csharp
/// <summary>
/// 保存面板数据到项目文件。
/// </summary>
/// <param name="projectFile">项目文件实例。</param>
public void SaveData(ProjectFile projectFile)
{
    if (projectFile != null)
    {
        projectFile.TextSplit_OriginalText = this.tb_原始文本.Text;
        projectFile.TextSplit_FinalText = this.tb_最终文本.Text;
        projectFile.TextSplit_SplitLength = (int)this.nud_拆分长度.Value;
    }
}
```

#### 3.2.4 修改 nud\_拆分长度\_ValueChanged 方法

```csharp
/// <summary>
/// 事件处理：拆分长度数值改变。
/// </summary>
private void nud_拆分长度_ValueChanged(object sender, EventArgs e)
{
    if (!_isLoading)
    {
        // 只有在非项目文件来源时，或者用户手动修改后，才保存到系统配置
        Setting.SetTextSplit_SplitLength(CurrentEngineId, (int)this.nud_拆分长度.Value);
        _isSplitLengthFromProject = false; // 用户已修改，标记为非项目文件来源
    }
}
```

        }

        {
            // ... 错误处理
        }
    }
}

## 五、注意事项

1. **向后兼容性**：使用可空类型 `int?` 存储拆分长度，旧项目文件中不存在该字段时返回 `null`，不影响加载逻辑。

2. **默认值**：默认拆分长度为 100，与现有代码保持一致。

3. **系统配置保存时机**：

   * 用户通过 UI 修改拆分长度时，立即保存到系统配置

   * 从项目文件加载时，不保存到系统配置

   * 这确保了系统配置始终保存"最后一次修改过的值"

4. **忽略换行符设置**：本次需求未提及"忽略换行符"设置，保持现有逻辑不变（始终从系统配置加载）。

5. **软件启动与新建的区别**：

   * 软件启动时选择"是/否/取消"：都从系统配置加载拆分长度

   * 主菜单"新建"：保持UI现有拆分长度值不变
