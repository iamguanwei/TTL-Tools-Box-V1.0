# 默认朗读者保存机制改造方案

## 一、现状分析

### 1.1 当前实现

* **保存位置**：已经保存在项目文件中（`EngineProjectData.VoicePreprocess_DefaultSpeaker`）

* **问题**：

  * 没有处理"已丢失"的情况，找不到时直接选第一个

  * TTL引擎未连接时下拉列表为空，没有显示"已丢失"状态

  * "发送到语音生成"按钮没有检查选中的朗读者是否有效

### 1.2 参考实现（角色映射源名称）

* 使用"（已丢失）"后缀标记不存在的朗读者

* 有 `IsLost` 属性标记状态

* 下拉列表存储字符串类型，支持添加"已丢失"项

***

### 2.2 UI层改造

#### 2.2.1 修改 load语音生成预处理默认角色设置() 方法

**当前逻辑**：

```
清空下拉列表 → 添加引擎朗读者 → 匹配保存值 → 找不到则选第一个
```

**改造后逻辑**：

```
清空下拉列表 → 添加引擎朗读者 → 匹配保存值
  → 找到：选中该项，_isDefaultSpeakerLost = false
  → 未找到且保存值不为空：添加"保存值（已丢失）"并选中，_isDefaultSpeakerLost = true
  → 保存值为空且有朗读者：选第一个，_isDefaultSpeakerLost = false
  → 保存值为空且无朗读者：保持空白，_isDefaultSpeakerLost = false
```

#### 2.2.2 新增 refresh默认朗读者Options() 方法

参考 `RoleMappingPanel.refresh角色映射SourceNameOptions()` 实现：

```csharp
/// <summary>
/// 刷新默认朗读者下拉列表选项。
/// </summary>
private void refresh默认朗读者Options()
{
    this.cb_语音生成预处理_默认朗读者设置.Items.Clear();
    _isDefaultSpeakerLost = false;

    // 获取当前可用的朗读者列表
    HashSet<string> availableSourceNames = new HashSet<string>();
    var currentEngine = ConnectionManager?.CurrentEngine;
    if (currentEngine != null)
    {
        var speakers = currentEngine.Speakers;
        if (speakers != null && speakers.Length > 0)
        {
            foreach (var speaker in speakers)
            {
                this.cb_语音生成预处理_默认朗读者设置.Items.Add(speaker.SourceName);
                availableSourceNames.Add(speaker.SourceName);
            }
        }
    }

    // 获取保存的默认朗读者
    string savedSpeakerName = string.Empty;
    if (ProjectFile != null && !string.IsNullOrEmpty(CurrentEngineId))
    {
        savedSpeakerName = ProjectFile.GetVoicePreprocess_DefaultSpeaker(CurrentEngineId);
    }

    // 处理"已丢失"后缀
    string baseSpeakerName = savedSpeakerName;
    bool hasLostSuffix = !string.IsNullOrEmpty(savedSpeakerName) && savedSpeakerName.EndsWith(Constants.已丢失_标识后缀);
    if (hasLostSuffix)
    {
        baseSpeakerName = savedSpeakerName.Substring(0, savedSpeakerName.Length - Constants.已丢失_标识后缀.Length);
    }

    // 判断保存的朗读者是否可用
    if (!string.IsNullOrWhiteSpace(baseSpeakerName))
    {
        if (availableSourceNames.Contains(baseSpeakerName))
        {
            // 朗读者可用，选中该项
            this.cb_语音生成预处理_默认朗读者设置.SelectedItem = baseSpeakerName;
            _isDefaultSpeakerLost = false;
            
            // 如果之前保存的是带"已丢失"后缀的，更新为正常名称
            if (hasLostSuffix)
            {
                ProjectFile?.SetVoicePreprocess_DefaultSpeaker(CurrentEngineId, baseSpeakerName);
            }
        }
        else
        {
            // 朗读者不可用，添加"已丢失"项并选中
            string lostName = baseSpeakerName + Constants.已丢失_标识后缀;
            this.cb_语音生成预处理_默认朗读者设置.Items.Add(lostName);
            this.cb_语音生成预处理_默认朗读者设置.SelectedItem = lostName;
            _isDefaultSpeakerLost = true;
            
            // 更新保存的名称（添加"已丢失"后缀）
            ProjectFile?.SetVoicePreprocess_DefaultSpeaker(CurrentEngineId, lostName);
        }
    }
    else
    {
        // 没有保存值，使用默认逻辑
        if (this.cb_语音生成预处理_默认朗读者设置.Items.Count > 0)
        {
            this.cb_语音生成预处理_默认朗读者设置.SelectedIndex = 0;
        }
        _isDefaultSpeakerLost = false;
    }
}
```

### 2.3 事件处理

#### 2.3.1 下拉列表选中项变更事件

当用户手动选择一个有效的朗读者时：

* 更新 `_isDefaultSpeakerLost = false`

* 保存新的默认朗读者

* 刷新按钮状态

```csharp
private void cb_语音生成预处理_默认朗读者设置_SelectedIndexChanged(object sender, EventArgs e)
{
    string selectedSpeaker = this.cb_语音生成预处理_默认朗读者设置.SelectedItem as string;
    
    if (!string.IsNullOrWhiteSpace(selectedSpeaker) && !selectedSpeaker.EndsWith(Constants.已丢失_标识后缀))
    {
        _isDefaultSpeakerLost = false;
        
        if (ProjectFile != null && !string.IsNullOrEmpty(CurrentEngineId))
        {
            ProjectFile.SetVoicePreprocess_DefaultSpeaker(CurrentEngineId, selectedSpeaker);
        }
    }
    
    refresh语音生成预处理Ui();
}
```

### 2.4 引擎状态变化处理

#### 2.4.1 引擎连接状态变化时

调用 `refresh默认朗读者Options()` 刷新下拉列表。

#### 2.4.2 切换引擎时

在 `LoadData(string engineId)` 方法中调用 `refresh默认朗读者Options()`。

## 三、边界情况处理

| 场景         | 处理方式                          |
| ---------- | ----------------------------- |
| TTL引擎未连接   | 下拉列表只显示"已丢失"项（如果有保存值），否则为空    |
| TTL引擎无朗读者  | 下拉列表为空，按钮不可用                  |
| 保存的朗读者已删除  | 显示"名称（已丢失）"并选中，按钮不可用          |
| 保存的朗读者恢复   | 自动移除"已丢失"后缀，恢复正常选中状态          |
| 用户选择"已丢失"项 | 不允许（下拉列表中的"已丢失"项仅用于显示，不可主动选择） |

## 六、补充考虑

### 6.1 数据保存格式

* 保存时保存完整的字符串（包含"已丢失"后缀）

* 这样可以在项目文件中清楚地看到哪些配置有问题

* 当朗读者恢复时自动移除后缀

### 6.2 与角色映射的一致性
- **统一使用已存在的常量**：`Constants.已丢失_标识后缀`（定义在 `Constants.cs:64`）
- 处理逻辑与角色映射保持一致
- 该常量已在 `RoleMappingPanel.cs` 中使用，本次改造只需在 `VoicePreprocessPanel.cs` 中引用同一常量

### 6.3 用户体验

* "已丢失"项在下拉列表中可见但不应让用户主动选择

* 当用户选择其他有效朗读者时，自动清除"已丢失"状态
