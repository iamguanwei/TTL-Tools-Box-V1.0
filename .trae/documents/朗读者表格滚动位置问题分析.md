# 朗读者表格滚动位置问题完整分析

## 问题现象

点击"还原"按钮后，表格滚动位置没有保持。

## 完整调用链分析

### 还原按钮调用链

    │

    │
    ├─ 第1272行: createSpeakersFromRoles(ttl)

## 问题根源

**问题1：`setupSpeakerGridColumns` 被重复调用**

在 `cb_TTL方案_当前方案名称_SelectedIndexChanged` 中：
- 第1272行 `createSpeakersFromRoles(ttl)` 内部已经调用了 `setupSpeakerGridColumns()`
- 第1273行又显式调用了 `setupSpeakerGridColumns()`

**问题2：`setupSpeakerGridColumns` 中的 `Columns.Clear()` 会重置滚动位置**

```csharp
// setupSpeakerGridColumns 第997行
this.dgv_TTL方案_朗读者参数配置.Columns.Clear();
```

当 `Columns.Clear()` 被调用时，DataGridView 的内部状态会被重置，包括滚动位置。

**时序问题：**
1. `bindSpeakersToGrid` 中恢复了滚动位置
2. 紧接着 `cb_TTL方案_当前方案名称_SelectedIndexChanged` 第1273行又调用了 `setupSpeakerGridColumns()`
3. `setupSpeakerGridColumns` 中的 `Columns.Clear()` 重置了滚动位置

## 解决方案

### 方案：移除多余的 `setupSpeakerGridColumns()` 调用

在 `cb_TTL方案_当前方案名称_SelectedIndexChanged` 中，第1273行的 `setupSpeakerGridColumns()` 是多余的，因为 `bindSpeakersToGrid` 中已经调用过了。

**修改位置：** [TtlSchemePanel.cs:1272-1274](file:///d:/User%20Data/guanw@hotmail.com/我的私有云/GW's%20Data/我的开发/GWs/TTL工具箱/TTL%20Tools%20Box%20V1.0/GW.TTLtoolsBox.WinFormUi/UI/Panels/TtlSchemePanel.cs#L1272-L1274)

**修改前：**
```csharp
createSpeakersFromRoles(ttl);
setupSpeakerGridColumns();
```

**修改后：**
```csharp
createSpeakersFromRoles(ttl);
```
