# "插队"菜单项禁用问题分析与修复方案

## 一、问题分析

### 1.1 当前代码逻辑

```csharp
if (isSingleSelection)
{
    VoiceGenerationTask singleTask = selectedTasks[0];
    bool isQueued = singleTask.Status == VoiceGenerationTaskStatus.排队中;
    bool isFirstQueued = true;

    if (isQueued)
    {
        foreach (var t in _voiceGenerationTaskQueue.Tasks)
        {
            if (t.Status == VoiceGenerationTaskStatus.排队中)
            {
                if (t != singleTask)
                {
                    isFirstQueued = false;
                }
                break;
            }
        }
    }

    插队IToolStripMenuItem.Enabled = isQueued && !isFirstQueued;
}
```

### 1.2 问题根因

代码使用 `t != singleTask` 进行对象引用比较。问题在于：

**`selectedTasks[0]` 获取的任务对象和 `_voiceGenerationTaskQueue.Tasks` 中的任务对象可能不是同一个引用！**

虽然 DataGridView 的数据源是 `_voiceGenerationTaskQueue.Tasks`，但在某些情况下（如数据刷新、重新绑定等），`row.DataBoundItem` 返回的对象可能与原列表中的对象引用不同。

### 1.3 验证方式

应该使用任务ID进行比较，而不是对象引用比较：

```csharp
if (t.Id != singleTask.Id)  // 使用ID比较
```

## 二、修复方案

### 修改位置

文件：`VoiceGenerationPanel.cs`
方法：`cms_语音生成_任务控制_Opening`

### 修改内容

将对象引用比较改为任务ID比较：

**修改前：**
```csharp
if (t != singleTask)
{
    isFirstQueued = false;
}
```

**修改后：**
```csharp
if (t.Id != singleTask.Id)
{
    isFirstQueued = false;
}
```

## 三、完整修改代码

```csharp
if (isSingleSelection)
{
    VoiceGenerationTask singleTask = selectedTasks[0];
    bool isQueued = singleTask.Status == VoiceGenerationTaskStatus.排队中;
    bool isFirstQueued = true;

    if (isQueued)
    {
        foreach (var t in _voiceGenerationTaskQueue.Tasks)
        {
            if (t.Status == VoiceGenerationTaskStatus.排队中)
            {
                if (t.Id != singleTask.Id)
                {
                    isFirstQueued = false;
                }
                break;
            }
        }
    }

    插队IToolStripMenuItem.Enabled = isQueued && !isFirstQueued;
}
```

## 四、测试要点

1. 选择第一个"排队中"的任务 → 菜单应禁用
2. 选择非第一个"排队中"的任务 → 菜单应启用
3. 选择非"排队中"状态的任务 → 菜单应禁用
4. 选择多个任务 → 菜单应禁用
