# 语音生成文件名序号策略优化计划

## 一、需求分析

### 当前行为
语音生成保存文件时，无论任务数量多少，都会在文件名末尾自动添加数字序号（如 `file01.wav`、`file02.wav`）。

### 目标行为
1. **单个任务**：文件名不添加数字序号，直接使用用户输入的文件名（如 `file.wav`）
2. **多个任务**：保持现有策略，在文件名末尾添加数字序号（如 `file01.wav`、`file02.wav`...）

```csharp
{
    VoiceGenerationTask task = new VoiceGenerationTask();
```

```csharp
    task.SaveFile = Path.Combine(folderPath, $"{mFileName}{fileNameSuffix}{eFileName}");
```

## 三、逻辑说明

1. 新增 `isSingleTask` 变量判断是否为单个任务
2. 单个任务时，`fileNameSuffix` 为空字符串，文件名不添加数字
3. 多个任务时，保持原有逻辑，使用 `PadNumber` 生成序号

## 四、影响范围

- 仅影响语音生成任务的文件名生成逻辑
- 不影响其他功能模块
- 向后兼容，多任务场景行为不变
