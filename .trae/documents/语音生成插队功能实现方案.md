# 语音生成"插队"功能实现方案

## 一、需求概述

在"语音生成"表格的快捷菜单中添加"插队"功能，实现以下需求：

1. **功能目的**：让某个排队中的任务立即开始执行
2. **启用条件**：仅在选择一个任务，且该任务处于"排队中"状态时有效
3. **执行逻辑**：
   - 检查选定任务状态是否为"排队中"
   - 停止队列中正在执行的任务
   - 启动选定任务
4. **约束条件**：该操作不会改变任务顺序

## 二、技术分析

### 2.1 现有代码结构

#### 任务状态枚举
```csharp
public enum VoiceGenerationTaskStatus
{
    未开始,
    排队中,      // ← 关键状态
    正在生成,
    已完成,
    生成失败,
    转换MP3,
    转换MP3失败
}
```

#### 任务执行机制
队列管理器 `VoiceGenerationTaskQueue` 按以下逻辑执行任务：
1. 按 `_tasks` 列表顺序遍历
2. 查找第一个"排队中"状态的任务
3. 将其状态改为"正在生成"并执行

#### 关键字段
- `_currentTask`: 当前正在执行的任务
- `_isRunning`: 是否有任务正在执行
- `_shouldStop`: 停止标志

### 2.2 核心问题

队列按顺序执行第一个"排队中"的任务。要让某个非队首的"排队中"任务立即执行，需要引入"优先执行"机制。

### 2.3 解决方案

在 `VoiceGenerationTaskQueue` 中添加优先任务机制：
1. 新增 `_priorityTask` 字段存储需要优先执行的任务
2. 新增 `PrioritizeTask(VoiceGenerationTask task)` 方法
3. 修改 `ProcessNextTaskAsync` 逻辑，优先检查并执行 `_priorityTask`

## 三、实现步骤

### 步骤1：修改 VoiceGenerationTaskQueue.cs

#### 1.1 添加字段
在 `#region private` → `#region 字段` 区域添加：
```csharp
/// <summary>
/// 优先执行的任务。
/// </summary>
private VoiceGenerationTask _priorityTask = null;
```

#### 1.2 添加公共方法
在 `#region public` → `#region 方法` 区域添加：
```csharp
/// <summary>
/// 将指定任务设为优先执行。
/// </summary>
/// <param name="task">要优先执行的任务。</param>
/// <remarks>
/// 该方法会停止当前正在执行的任务，并将指定任务设为优先执行。
/// 任务顺序不会改变。
/// </remarks>
public void PrioritizeTask(VoiceGenerationTask task)
{
    if (task == null) return;
    if (task.Status != VoiceGenerationTaskStatus.排队中) return;
    
    // 停止当前任务
    if (_isRunning && _currentTask != null)
    {
        _shouldStop = true;
    }
    
    // 设置优先任务
    _priorityTask = task;
    
    // 如果队列未运行，启动队列
    if (!_isRunning)
    {
        ProcessNextTask();
    }
}
```

#### 1.3 修改 ProcessNextTaskAsync 方法
修改任务选择逻辑，优先检查 `_priorityTask`：

**原代码（第387-405行）**：
```csharp
VoiceGenerationTask nextTask = null;
foreach (var task in _tasks)
{
    if (task.Status == VoiceGenerationTaskStatus.排队中)
    {
        nextTask = task;
        break;
    }
}
```

**修改为**：
```csharp
VoiceGenerationTask nextTask = null;

// 优先执行插队任务
if (_priorityTask != null && _priorityTask.Status == VoiceGenerationTaskStatus.排队中)
{
    nextTask = _priorityTask;
    _priorityTask = null;
}
else
{
    _priorityTask = null;
    foreach (var task in _tasks)
    {
        if (task.Status == VoiceGenerationTaskStatus.排队中)
        {
            nextTask = task;
            break;
        }
    }
}
```

### 步骤3：修改 VoiceGenerationPanel.cs

#### 3.1 添加菜单启用状态控制
在 `cms_语音生成_任务控制_Opening` 方法中（约第556-648行），添加插队菜单项的启用状态控制：

在现有菜单项启用状态设置后添加：
```csharp
// 插队菜单项控制
插队IToolStripMenuItem.Enabled = false;
if (hasSelectedRows && selectedCount == 1)
{
    VoiceGenerationTask singleTask = selectedTasks[0];
    bool isQueued = singleTask.Status == VoiceGenerationTaskStatus.排队中;
    bool isFirstQueued = true;
    
    // 检查是否是第一个排队中的任务
    if (isQueued)
    {
        foreach (var t in _voiceGenerationTaskQueue.Tasks)
        {
            if (t.Status == VoiceGenerationTaskStatus.排队中)
            {
                if (t != singleTask)
                {
                    isFirstQueued = false;
                }
                break;
            }
        }
    }
    
    // 只有排队中且不是第一个排队任务时才启用
    插队IToolStripMenuItem.Enabled = isQueued && !isFirstQueued;
}
```

#### 3.2 添加菜单项Click事件处理方法
在 `#region 事件处理` 区域添加：
```csharp
/// <summary>
/// 事件处理：点击"插队"菜单项。
/// </summary>
private void 插队IToolStripMenuItem_Click(object sender, EventArgs e)
{
    if (dgv_语音生成_任务清单.SelectedRows.Count != 1)
    {
        return;
    }

    VoiceGenerationTask selectedTask = dgv_语音生成_任务清单.SelectedRows[0].DataBoundItem as VoiceGenerationTask;
    if (selectedTask == null)
    {
        return;
    }

    if (selectedTask.Status != VoiceGenerationTaskStatus.排队中)
    {
        MessageBox.Show("只能对\"排队中\"状态的任务执行插队操作。", "提示", MessageBoxButtons.OK, MessageBoxIcon.Information);
        return;
    }

    _voiceGenerationTaskQueue.PrioritizeTask(selectedTask);
}
```

## 四、执行流程图

```
用户点击"插队"菜单
        ↓
检查选中任务数量 = 1
        ↓
检查任务状态 = "排队中"
        ↓
调用 PrioritizeTask(task)
        ↓
设置 _shouldStop = true（停止当前任务）
        ↓
设置 _priorityTask = task
        ↓
当前任务执行循环检测到 _shouldStop
        ↓
当前任务状态改回"排队中"
        ↓
ProcessNextTask 检查 _priorityTask
        ↓
执行 _priorityTask（插队任务）
        ↓
执行完成后继续正常队列执行
```

## 五、边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 选中的任务已经是第一个排队任务 | 菜单项禁用 |
| 没有正在执行的任务 | 直接设置优先任务并启动队列 |
| 选中的任务状态不是"排队中" | 菜单项禁用 |
| 多个任务被选中 | 菜单项禁用 |
| 插队任务在等待执行时被删除 | `_priorityTask` 会被清空，不影响队列 |
