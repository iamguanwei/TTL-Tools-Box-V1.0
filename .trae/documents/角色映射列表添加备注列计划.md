# 角色映射列表添加"备注"列实施计划

## 一、需求概述
在角色映射列表中添加"备注"列，放在第3列（源名称之后、声音预览之前），最小宽度150，自动按最宽单元格扩展，只读。备注内容来自TTL方案中对应朗读者的备注信息。

## 二、涉及文件
1. `GW.TTLtoolsBox.WinFormUi\UI\Panels\RoleMappingPanel.cs` - 角色映射面板

## 三、实施步骤

### 步骤1：修改setup角色映射GridColumns()方法
**文件：** `GW.TTLtoolsBox.WinFormUi\UI\Panels\RoleMappingPanel.cs`

**修改内容：**
在源名称列之后、声音预览列之前添加"备注"列：
- 名称：`Remark`
- 标题：`备注`
- 最小宽度：150
- 只读：true
- 自动调整：`AutoSizeMode = AllCells`

### 步骤2：实现备注数据的获取和显示
由于备注存储在`SpeakerInfo.Remark`中，需要通过`SourceName`关联获取：
- 在`CellFormatting`事件中，根据当前行的`SourceName`查找对应的`SpeakerInfo`
- 从`SpeakerInfo.Remark`获取备注值并显示

### 步骤3：调整列顺序
确保列顺序为：
1. 角色名（可编辑）
2. 源名称（下拉选择）
3. **备注（只读）** ← 新增
4. 声音预览（按钮）

## 四、数据关联机制
```
角色映射列表                    朗读者列表
┌─────────────────┐            ┌─────────────────┐
│ RoleMappingItem │            │  SpeakerInfo    │
│  - RoleName     │            │  - SourceName   │
│  - SourceName ──┼───────────►│  - Speed        │
│                 │    匹配     │  - Volume       │
│                 │            │  - Remark ──────┼──► 显示到备注列
└─────────────────┘            └─────────────────┘
```

## 五、注意事项
1. 备注列是只读的，不需要修改`RoleMappingItem`实体类
2. 备注数据来自当前TTL引擎的`SpeakerInfo`列表
3. 如果找不到对应的`SpeakerInfo`或备注为空，显示空字符串
