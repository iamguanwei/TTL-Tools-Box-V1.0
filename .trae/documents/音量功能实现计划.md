# 音量功能实现计划

## 一、概述

参照现有Speed（语速）功能的实现方式，为任务、任务项和朗读者添加"音量"设置功能。

### 音量处理逻辑（与Speed一致）
- **朗读者级别**：每个朗读者有自己的默认音量（50-200%，默认100%）
- **任务级别**：全局音量，0表示使用各自朗读者的音量
- **任务项级别**：单个任务项的音量
- **处理方式**：
  - 全局音量 > 0：先合并所有音频，再统一调整音量
  - 全局音量 = 0：先调整各段音频音量（使用各自朗读者的Volume），再合并

   ```csharp
   private void bt_TTL方案_清理无效的预览音频_Click(object sender, EventArgs e)
   {
       // 获取当前引擎ID
       string engineId = getCurrentEngineId();
       
       // 获取当前朗读者列表中的所有预览文件名模式
       var currentEngine = TtlSchemeController?.CurrentEngineConnector;
       if (currentEngine == null || currentEngine.Speakers == null) return;
       
       HashSet<string> validPatterns = new HashSet<string>();
       foreach (var speaker in currentEngine.Speakers)
       {
           // 格式：{engineId}_{sourceName}_{speed}_{volume}.mp3
           string safeSourceName = getSafeFileName(speaker.SourceName);
           validPatterns.Add($"{engineId}_{safeSourceName}_{speaker.Speed}_{speaker.Volume}.mp3");
       }
       
       // 遍历预览文件夹，删除不在validPatterns中的文件
       string previewFolder = TTL角色预览声音_文件夹路径;
       if (Directory.Exists(previewFolder))
       {
           var files = Directory.GetFiles(previewFolder, $"{engineId}_*.mp3");
           foreach (var file in files)
           {
               string fileName = Path.GetFileName(file);
               if (!validPatterns.Contains(fileName))
               {
                   try { File.Delete(file); } catch { }
               }
           }
       }
   }
   ```

## 四、ffmpeg音量调整命令

音量调整使用ffmpeg的volume滤镜：

```bash
ffmpeg -i input.wav -filter:a "volume=1.5" output.wav
```

其中：
- volume参数 = 音量百分比 / 100
- 100%音量 = volume=1.0（不调整）
- 150%音量 = volume=1.5
- 50%音量 = volume=0.5
