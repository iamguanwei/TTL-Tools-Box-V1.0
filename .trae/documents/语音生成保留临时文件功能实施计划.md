# 实施计划：语音生成面板"保留临时文件"功能

## 需求概述
在语音生成面板添加的"cb_语音生成_保留临时文件"复选框需要完善以下功能：
1. 复选框状态保存在系统配置中，启动软件时自动读取，默认值不选中
2. 选中时，任务完成后不删除原始语音临时文件（temp_开头的文件）；不选中时照常删除

## 技术分析

### 临时文件类型
根据代码分析，项目中存在以下临时文件：
- **原始语音临时文件**：`temp_{task.Id}_{i+1}.wav` - 用户要求保留的文件
- 合并后临时文件：`merged_{Guid}.wav`
- 调速后临时文件：`speed_{Guid}.wav`
- MP3转换临时文件：`temp_mp3_{Guid}.mp3`

### 配置保存方式
项目使用 `Setting.GetValue()` 和 `Setting.SetValue()` 方法读写配置，配置保存在 `Setting.ini` 文件中。

### 现有示例
复选框状态保存可参考 `cb_语音生成_自动开始任务` 的实现方式。

## 实施步骤

### 步骤1：VoiceGenerationPanel.cs - 添加复选框状态保存和读取

**位置**：`init语音生成Ui()` 方法中，在加载"自动开始任务"复选框状态的代码之后

**操作**：
1. 添加读取复选框状态的代码：
```csharp
// 加载"保留临时文件"复选框状态
bool keepTempFiles = bool.Parse(Setting.GetValue(nameof(cb_语音生成_保留临时文件), false.ToString()));
cb_语音生成_保留临时文件.Checked = keepTempFiles;
```

2. 添加 `CheckedChanged` 事件处理程序：
```csharp
/// <summary>
/// 事件处理：保留临时文件复选框状态变更。
/// </summary>
private void cb_语音生成_保留临时文件_CheckedChanged(object sender, EventArgs e)
{
    Setting.SetValue(nameof(cb_语音生成_保留临时文件), cb_语音生成_保留临时文件.Checked.ToString());
}
```

3. 在 Designer.cs 中绑定事件处理程序

### 步骤2：VoiceGenerationTaskQueue.cs - 添加保留临时文件设置支持

**操作**：
1. 添加属性用于获取保留临时文件的设置：
```csharp
/// <summary>
/// 获取或设置是否保留原始语音临时文件。
/// </summary>
public Func<bool> GetKeepTempFiles { get; set; }
```

2. 修改 `ExecuteTaskAsync` 方法中的临时文件清理逻辑（finally块），区分原始语音临时文件和其他临时文件：
   - 原始语音临时文件（temp_开头）：根据设置决定是否删除
   - 其他临时文件（merged_、speed_、temp_mp3_）：始终删除

### 步骤3：VoiceGenerationPanel.cs - 传递设置到任务队列

**位置**：`init语音生成Ui()` 方法中，在创建 `_voiceGenerationTaskQueue` 之后

**操作**：
添加获取保留临时文件设置的委托：
```csharp
_voiceGenerationTaskQueue.GetKeepTempFiles = () => cb_语音生成_保留临时文件.Checked;
```

## 涉及文件

| 文件 | 修改内容 |
|------|----------|
| VoiceGenerationPanel.cs | 添加复选框状态读取、保存和传递逻辑 |
| VoiceGenerationPanel.Designer.cs | 绑定 CheckedChanged 事件 |
| VoiceGenerationTaskQueue.cs | 添加保留临时文件属性，修改清理逻辑 |

## 注意事项
1. 默认值为 `false`（不选中），即默认删除临时文件
2. 只保留原始语音临时文件（temp_开头），其他临时文件始终删除
3. 任务被删除时，临时文件仍然需要删除（不受此设置影响）
